「**Data install**」意指為[PlayStation
Portable遊戲資料](../Page/PlayStation_Portable.md "wikilink")，安裝到「Memory
Stick Pro Duo」之上的技術，該技術比較為人熟識的名字為CAPCOM名下的「Media Install」，但實際上「Data
install」只是該等技術通稱，並沒有真正統一名字（原因後述）。

## 概述

[PSP平台遊戲](https://zh.wikipedia.org/wiki/PSP "wikilink")，因為媒體及[PSP](https://zh.wikipedia.org/wiki/PSP "wikilink")(PSP-1000/2000/3000機型)機能所限（主要為UMD光驅），向來有讀取過慢的惱人問題。一般而言，冗長的讀取時間會破壞玩家的興致。雖然部份開發廠商透過開發手段解決問題，如透過計算遊戲資料的讀取頻率，調整UMD光碟刻錄資料位置，以減少光頭移動及尋找資料頻率；以及索尼開發新型PSP，透過增加[UMD光驅快取減少讀取UMD次數](../Page/通用媒體光碟.md "wikilink")，但只能緩和[UMD讀取速度問題](../Page/通用媒體光碟.md "wikilink")。

隨著Memory Stick容量及普及性發展，有廠商想到以利用[Memory
Stick加快資料讀取的手段](https://zh.wikipedia.org/wiki/Memory_Stick "wikilink")。Data
install 用意，透過遊戲程式，先從UMD媒體下載相關的預讀資料到搭載PSP的[Memory Stick
Duo之上](https://zh.wikipedia.org/wiki/Memory_Stick#Memory_Stick_PRO_Duo "wikilink")（以遊戲“Save
Data”形式記錄），遊戲程式能藉此令讀取時間短縮，大幅強化原來遊戲只依靠UMD的讀取速度。一般預讀資料大小，由 64MB到 1GB
以上不等。一般預讀資料多預載語音或模型貼圖，一類頻繁讀取又大容量的資料。形式而言，狀況跟往年電腦遊戲，預先安裝遊戲資料到電腦硬碟上作法差不多。

此外，該等技術是針對擁有UMD光驅的PSP。由於新型PSP
Go(PSP-N1000型)只使用內存快閃記憶體儲存遊戲，對比使用UMD版的舊型號機，讀取效率更高，故無需使用
Data Install機能。

## 發展

據記錄，最先採用這種形式的遊戲為於2007年末發售的移植遊戲《[AIR](../Page/AIR_\(遊戲\).md "wikilink")》，當時預讀資料量為64
MB-120 MB不等。Prototype旗下公司稱該機能為 「**Memory Stick
機能**」。後來，[CAPCOM的](https://zh.wikipedia.org/wiki/CAPCOM "wikilink")《[魔物獵人攜帶版2nd
G](../Page/魔物獵人攜帶版2nd_G.md "wikilink")》亦推出命為「Media
Install」的類似機能，後來更將有關名字註冊成商標。\[1\]從此以後，陸續有不同遊戲廠商，陸續將類似功能載入到遊戲之上增加競爭力，而名字上也傾向採用
「Data Install」。

最近，Prototype 宣布移植遊戲《[Little Buster \! Converted
Edition](../Page/Little_Busters!.md "wikilink")》的PSP版，於新聞稿表示，該遊戲兩片組的UMD中的一片UMD為安裝光碟。其目的，是為了不破壞PSP便攜性為旨，無須轉換光碟片亦能享受完整的遊戲內容。\[2\]

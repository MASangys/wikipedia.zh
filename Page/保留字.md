**保留字**（），有时也叫**关键字**（）\[1\]\[2\]，是[编程语言](../Page/编程语言.md "wikilink")中的一类语法结构。在特定的编程语言里，这些保留字具有较为特殊的意义，并且在语言的格式说明里被预先定义。通常，保留字包括用来支持[类型系统的原始数据类型的](https://zh.wikipedia.org/wiki/类型系统 "wikilink")[标记](../Page/標記_\(程式語言\).md "wikilink")，并可以用来识别诸如循环结构、语句块、条件、分支等程序结构。

保留字在编程语言设计之时就被定义。有时，语言说明具有一定的灵活性，软件商可能会在其生产的[编译器中添加非标准的语言特性](https://zh.wikipedia.org/wiki/编译器 "wikilink")。同样，当发展得更加成熟时，编程语言的保留字标准可能会扩展一些额外的特性，例如，在传统的编程语言中增加[面向对象程序设计](../Page/面向对象程序设计.md "wikilink")的特性。有时，编程语言说明会包含一些将来版本可能需要用到的保留字。在[Java](../Page/Java.md "wikilink")中，`const`和`goto`就是两个保留字——它们在Java中目前没有被使用，因此不具有意义，但是不能够被用作标识符。\[3\]通过“保留”这个术语，它们可以在Java的未来版本中补充，而不需要“破坏”旧的Java源代码。不像预定义函数、方法和子程序，保留字不能被程序员定义，而前面那些的名称通常被归类于标识符，而不是保留字。

## 语言间的比较

不是所有的编程语言都有相同数量的保留字。例如，Java（以及其他[C语言](https://zh.wikipedia.org/wiki/C语言 "wikilink")\[4\]派生的语言），具有相对较少的保留字，大约有50个。而[COBOL](../Page/COBOL.md "wikilink")则具有大约400个保留字。相反的极端则是[Prolog](../Page/Prolog.md "wikilink")，根本就没有保留字。

一个语言具有的保留字数量与其功能“强大”程度没有必然联系。COBOL在20世纪50年代被设计成一种商业语言，它是一种自存档（）、使用类似[英语](../Page/英语.md "wikilink")结构的元素（例如动词、子句、语句等）。而相反地，C语言被设计成一种在句法上较为简练的编程语言。例如，比较下列来自C语言和COBOL的两段计算单周收入的等价语句块：

``` c
// 使用C语言计算：

if (salaried)
        amount = 40 * payrate;
else
        amount = hours * payrate;
```

``` fortran

* 使用COBOL计算：

IF Salaried THEN
        MULTIPLY Payrate BY 40 GIVING Amount
ELSE
        MULTIPLY Payrate BY Hours GIVING Amount
END-IF.
```

``` fortran

* 另一个使用COBOL计算的例子：

IF Salaried
        COMPUTE Amount = Payrate * 40
ELSE
        COMPUTE Amount = hours * payrate
END-IF.
```

上述例子中彩色的部分即是它们各自的保留字。

纯的Prolog语言使用关系式来表达，并通过询问这些关系触发处理方法。诸如循环的结构通过使用循环、递归的关系来实现。

虽然以上三個例子具有不同的保留字，但是它们都能胜任解决上述问题。程序的功能取决于它们对一系列[图灵语言的归属](https://zh.wikipedia.org/wiki/图灵完全 "wikilink")。

## 一些跨语言的保留字

微软[.NET框架](../Page/.NET框架.md "wikilink")下的[通用语言架构](../Page/通用语言架构.md "wikilink")允许程序源代码以超过40种不同的编程语言混合编译，并生成一个解决方案。鉴于此，标识符、保留字的冲突可能会在一种语言写成的源代码试图处理另一种语言的源代码的过程中发生。例如，一个[Visual Basic .NET库可能会包含这样一个类](../Page/Visual_Basic_.NET.md "wikilink")：

``` vbnet
' Class Definition of This in Visual Basic.NET:

Public Class this
        ' This class does something...
End Class
```

假设这一段源代码被编译，并被作为一个工具箱的一部分，那么一个[C\#](https://zh.wikipedia.org/wiki/C_Sharp "wikilink")\[5\]程序员在同一程序中定义`this`类的变量就会遇到问题，因为`this`是C\#中的一个保留字，因此，下列的源代码将不能成功编译：

``` csharp
// Using This Class in C#:

this x = new this();  // Won't compile!
```

类似的问题在访问成员、覆盖虚方法以及识别[命名空间](../Page/命名空间.md "wikilink")的情况中出现。

为了解决这个问题，语言说明允许C\#程序员在保留字前面加上符号“[@](../Page/@.md "wikilink")”，从而强制编译器将后面的保留字看成一般的标识符，而非保留字。\[6\]例如：

``` csharp
// Using This Class in C#:

@this x = new @this();  // Will compile!
```

相反地，也可在 VB 中使用 \[\] 括起来以达到相似的目的：

``` csharp
//C#:
public class Me{
...
}
```

``` vbnet
//VB:
Dim myMe As New [Me]()
```

为了保持连贯性，这样的用法也可以在非公有情况下使用，例如声明[局部变量](https://zh.wikipedia.org/wiki/局部变量 "wikilink")、参数名称以及私有成员。

另一例子是 SQL 中也可用 \[\] 括起数据库对象名（表、列等），而被括的不仅限于 SQL 保留字 / 关键字：

``` sql
select [1st col] from [select]
```

这将选出名为“select”表中“1st col”列的数据。

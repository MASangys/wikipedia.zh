**克里斯托菲德斯算法 (Christofides algorithm)** 是[旅行商问题在](../Page/旅行推销员问题.md "wikilink")[度量空间](../Page/度量空间.md "wikilink")（即距离对称且满足[三角不等式](../Page/三角不等式.md "wikilink")）上的一个[近似算法](../Page/近似算法.md "wikilink")。\[1\] 该算法可以保证相对最优哈密尔顿回路长度有3/2的近似比。尼科斯·克里斯托菲德斯 (Nicos Christofides) 于1976年首次发表了这个算法，故以他的名字命名之。\[2\] ，这一算法仍然是一般性旅行商问题的算法中近似比最好的结果。

## 算法

令 <span class="texhtml">*G* = (*V*,*w*)</span> 是旅行商问题的一个实例。 即， <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">G</span> 是一顶点集<span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">V</span> 上的一个完全图，函数 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">w</span> 给 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">G</span> 的每条边指定了一个非负实边权。 依三角不等式，对每三个顶点 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">u</span>, <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">v</span> 和 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">x</span> 都有关系 <span class="texhtml">*w*(*uv*) + *w*(*vx*) ≥ *w*(*ux*)</span>.

算法可以被描述为如下[伪代码](../Page/伪代码.md "wikilink")。

1.  构造图上的[最小的生成树](../Page/最小生成树.md "wikilink") T
2.  令 O 为原图顶点集中在 T 上度数为奇数的顶点集。根据握手定理，O 中有偶数个顶点
3.  构造点集O在原完全图上的最小完美匹配M
4.  将 M 和 T 的边集取并，构造[重图](https://zh.wikipedia.org/wiki/伪图 "wikilink") H ，满足其每个顶点都是偶数度的
5.  H 可以形成一个[欧拉回路](https://zh.wikipedia.org/wiki/欧拉回路 "wikilink")
6.  将前一步得到的图改造为一个哈密尔顿回路：只需要跳过前一步欧拉回路中重复的顶点即可（这个步骤又称作选取近道，"short-cutting"）。

## 近似比

算法所生成的解相对最优解有3/2的近似比。下给出简要证明。

令 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 为图上的最优旅行商回路。注意到，删除 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 上的一条边将产生原图上的一棵生成树，其总权重至少为最小生成树的总权重，亦即 <span class="texhtml">*w*(*T*) ≤ *w*(*C*)</span>。接下来，给<span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">O</span>中的顶点编号，编号顺序依据回路<span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 上的顶点顺序。再将 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 做划分，得到两个边集：其一包含所有起始点为奇数编号的边，另一个则包含所有起始点为偶数编号的边。这两个边集各对应于 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">O</span> 上的一个完美匹配，且匹配的总边权不超过边集的总边权。

因为两个边集是 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 的划分，二者中的某一个至多有 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 总边权的一半，因而其对应匹配的总边权不超过 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">C</span> 总边权的一半。因为没有比最小完美匹配更小的匹配，所以 <span class="texhtml">*w*(*M*) ≤ *w*(*C*)/2</span>。 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">T</span> 和 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">M</span> 的权重总和也就是欧拉回路的权重总和，也就是至多 <span class="texhtml">3*w*(*C*)/2</span>。由于做选取近道操作并不增加权重，所有最终输出的总边权也不超过 <span class="texhtml">3*w*(*C*)/2</span>.

## 下界

存在使得克里斯托菲德斯算法近似比任意接近3/2的输入。

一类满足条件的输入由 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">n</span> 个顶点组成，满足最优环路上边的权重都为 <span class="texhtml">1</span>，且在最优环路上相隔一条边的边都有权重 <span class="texhtml">1 + *ε*</span>，其中 <span class="texhtml">*ε*</span> 是一个足够小的正数。未由上述定义给出的权重由两点间的[最短路径给出](../Page/最短路问题.md "wikilink")。

最小生成树可以从最优环路中构造，总长 <span class="texhtml">*n* − 1</span>。唯二的奇数度点是这棵树（此时也是条路径）的两个端点，其完美匹配仅由一条权重近似为 <span class="texhtml">*n*/2</span> 的单边构成。 生成树和这条单边的并集是一个回路，没有可能的近道 (short-cutting)，且其总边权近似为 <span class="texhtml">3*n*/2</span>。 而问题的最优解是使用所有权重为1或 <span class="texhtml">1 + *ε*</span> 的边构造的环路，其总权重为 <span class="texhtml">*n* + (*n* − 2)*ε*</span>，对充分小的 <span class="texhtml">*ε*</span> 可以无限靠近于 <span class="texhtml mvar" style="font-style:italic;margin-left:2px;margin-right:2px;">n</span>。\[3\]

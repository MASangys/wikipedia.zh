**全局描述符表** **(GDT)** 是一个从 [80286](../Page/Intel_80286.md "wikilink") 开始用于界定不同内存区域的特征，用于 [Intel](../Page/英特尔.md "wikilink") [x86](https://zh.wikipedia.org/wiki/x86 "wikilink")-系列处理器的数据结构。 全局描述表位于内存中。全局描述表的条目描述及规定了不同内存分区的各种特征，包括基地址、大小和访问等特权如可执行和可写等。 在 Intel 的术语中，这些内存区域被称为 *[段](../Page/X86記憶體區段.md "wikilink")* 。

全局描述表用于[内存地址的转换](https://zh.wikipedia.org/wiki/内存地址 "wikilink")。所有程序的内存访问都需要用到GDT中的有关内存区域即[x86内存分段的信息](https://zh.wikipedia.org/wiki/x86内存分段 "wikilink")。访问GDT需要使用segment selector和segment offset。处理器使用segment selector为索引查找GDT的条目。当适当的条目找到后，处理器将会做一系列的检查，包括检查segment offset尝试访问区间是否在此内存分段内，代码是否有权限访问此内存分段(检查[分级保护域](../Page/分级保护域.md "wikilink")权限）等。\[1\]

为了加速全局描述表的访问，往segment寄存器里载入segment的值会使得GDT的特定条目被读入处理器内部的缓存中。之后的内存访问将会通过缓存进行处理。

[SegmentDescriptor.svg](https://zh.wikipedia.org/wiki/File:SegmentDescriptor.svg "fig:SegmentDescriptor.svg")



## GDT作用

GDT的是提供内存保护。在[80286之前的处理器中只有](https://zh.wikipedia.org/wiki/80286 "wikilink")[实模式](https://zh.wikipedia.org/wiki/实模式 "wikilink")，所有程序都可访问任意内存。GDT是保护模式下限制非法内存访问的一种方式。

## GDT 结构 

### GDT description structure

[<File:GDTR结构.png>](https://zh.wikipedia.org/wiki/File:GDTR结构.png "fig:File:GDTR结构.png")

GDT description structure描述了GDT的位置及大小（并非GDT的一部分）。储存于GDTR 寄存器中。可通过LGDTR指令从内存中往GDTR中加载GDT description structure。

  - Size: GDT的大小（bytes) - 1
  - Offset: GDT的线性地址

## GDT实例

``` asm numberLines
gdt:
    .long 0
    .long 0
code_seg:
    .long 0x0000FFFF
    .long 0x00CF9A00
data_seg:
    .long 0x0000FFFF
    .long 0x00CF9200
gdt_ptr:
    .short 0x100
    .long gdt
```

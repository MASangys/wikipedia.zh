在[计算机科学中](../Page/计算机科学.md "wikilink")，**分治法**是建基於多項分支[遞歸的一种很重要的算法](https://zh.wikipedia.org/wiki/遞歸 "wikilink")[範式](https://zh.wikipedia.org/wiki/範式 "wikilink")。字面上的解释是“分而治之”，就是把一个复杂的问题分成两个或更多的相同或相似的子问题，直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并。

这个技巧是很多高效算法的基础，如[排序算法](../Page/排序算法.md "wikilink")（[快速排序](../Page/快速排序.md "wikilink")、[归并排序](../Page/归并排序.md "wikilink")）、[傅立叶变换](https://zh.wikipedia.org/wiki/傅立叶变换 "wikilink")（[快速傅立叶变换](https://zh.wikipedia.org/wiki/快速傅立叶变换 "wikilink")）。

另一方面，理解及設計分治法算法的能力需要一定時間去掌握。正如以歸納法去證明一個[理論](../Page/理論.md "wikilink")，為了使遞歸能夠推行，很多時候需要用一個較為概括或複雜的問題去取代原有問題。而且並沒有一個系統性的方法去適當地概括問題。

**分治法**這個名稱有時亦會用於將問題簡化為只有一個細問題的算法，例如用於在已排序的列中尋找其中一項的[折半搜索算法](https://zh.wikipedia.org/wiki/折半搜索算法 "wikilink")（或是在[數值分析中類似的](https://zh.wikipedia.org/wiki/數值分析 "wikilink")[勘根算法](https://zh.wikipedia.org/wiki/勘根定理 "wikilink")）。這些算法比一般的分治算法更能有效地執行。其中，假如算法使用[尾部遞歸的話](https://zh.wikipedia.org/wiki/尾部遞歸 "wikilink")，便能轉換成簡單的[迴圈](https://zh.wikipedia.org/wiki/程式迴圈 "wikilink")。但在這廣義之下，所有使用遞歸或迴圈的算法均被視作「分治算法」。因此，有些作者考慮「分治法」這個名稱應只用於每個有最少兩個子問題的算法。而只有一個子問題的曾被建議使用**減治法**這個名稱。

分治算法通常以[數學歸納法來驗證](https://zh.wikipedia.org/wiki/數學歸納法 "wikilink")。而它的計算成本則多數以解[遞迴關係式來判定](../Page/遞迴關係式.md "wikilink")。

## 早期历史上的先例

折半搜索算法——一個將原來問題連逐地拆細成大約一半大小的單一子問題的分治算法——擁有一段悠長歴史。雖然算法在計算機上的清楚描述出現在1946年約翰莫齊利（John
Mauchly）的一篇文章裡，然而利用已排序的物件序列去加快搜尋的構想早已在公元前200年的[巴比倫尼亞出現](https://zh.wikipedia.org/wiki/巴比倫尼亞 "wikilink")。另一個單一子問題的分治算法是找出2個數的[最大公因數的](../Page/最大公因數.md "wikilink")[輾轉相除法](../Page/輾轉相除法.md "wikilink")（透過將數字化小至使子問題變得簡單），於公元前數世紀已經出現。

一個早期有多個子問題的分治算法是[高斯在](../Page/卡爾·弗里德里希·高斯.md "wikilink")1805年描述關於快速傅立葉奱换的算法，儘管他沒有量化地分析它的[操作數目](../Page/計算複雜性理論.md "wikilink")，而快速傅立葉奱换直至在一世紀之後被重新發現之前亦沒有廣泛流傳。這個算法現在稱為[库利－图基快速傅里叶变换算法](../Page/库利－图基快速傅里叶变换算法.md "wikilink")。

至於專門用於計算機之上而且正確地分析的分治算法早期例子，則可以數到[约翰·冯·诺伊曼於](../Page/约翰·冯·诺伊曼.md "wikilink")1945年發明的歸並排序。

另一個顯著的例子是Anatolii Alexeevitch
Karatsuba於1960年發明在\(O(n^{\log_2 3})\)步驟內將兩個n位數相乘的[Karatsuba算法](../Page/Karatsuba算法.md "wikilink")。它反證了[安德雷·柯爾莫哥洛夫於](../Page/安德雷·柯爾莫哥洛夫.md "wikilink")1956年認為這個乘法需要\(\Omega(n^2)\)步驟的猜想。

[高德納舉了一個最初並沒有涉及計算機的分治算法例子](https://zh.wikipedia.org/wiki/高德納 "wikilink")，就是一般[郵局用於分發信件的方法](https://zh.wikipedia.org/wiki/郵局 "wikilink")：信件在主要郵局根據不同的地理範圍而分到不同的袋裡，每個袋亦在運送到地區郵局時分到更小的袋裡，如是者直至信件被派發為止。這個方法與早於1929年的[打孔卡排序機所用的](../Page/打孔卡.md "wikilink")[基数排序相類同](../Page/基数排序.md "wikilink")。

## 优势

### 解决困难问题

分治算法是一个解决复杂问题的好工具，它可以把问题分解成若干个子问题，把子问题逐个解决，再组合到一起形成大问题的答案。比如，汉诺塔问题如果采用分治算法，可以把高度为n的塔的问题转换成高度为n-1的塔来解决，如此重复，直至问题化简到可以很容易的处理为止。

### 算法效率

人们发现有很多效率很高的分治算法，比如，Karatsuba快速乘法算法、[快速排序算法和](../Page/快速排序.md "wikilink")[并行算法](https://zh.wikipedia.org/wiki/并行算法 "wikilink")、矩阵乘法的[施特拉森演算法](../Page/施特拉森演算法.md "wikilink")、[快速傅里叶变换等](../Page/快速傅里叶变换.md "wikilink")。

## 实现

### 循环递归

在每一层递归上都有三个步骤：

1.  分解：将原问题分解为若干个规模较小，相对独立，与原问题形式相同的子问题。
2.  解决：若子问题规模较小且易于解决时，则直接解。否则，递归地解决各子问题。
3.  合并：将各子问题的解合并为原问题的解。

## 示例

分治法在高级语言中主要的一个思想是[递归](../Page/递归.md "wikilink")，[LISP语言中的体现出了极丰富的分治法](https://zh.wikipedia.org/wiki/LISP语言 "wikilink")。

以下是归并排序[C语言的示例代码](https://zh.wikipedia.org/wiki/C语言 "wikilink")，输入参数中，需要排序的数组为array\[\],起始索引为first，终止索引为last。调用完成后，array\[\]中从first到last处于升序排列。

``` c
 void merge_sort(int array[], unsigned int first, unsigned int last)
 {
    int mid = 0;
    if(first<last)
    {
        mid = (first+last)/2;
        merge_sort(array, first, mid);
        merge_sort(array, mid+1,last);
        merge(array,first,mid,last);
    }
 }
```

在程式中可以看出分治法的應用：在merge_sort()中，將原來針對索引first到last的數組排序的問題，分為二份較小的問題

  - 先針對索引first到mid的數組排序。
  - 再針對索引mid+1到last的數組排序。

最後再進行二個數組的合併。

在[抽象代数中](../Page/抽象代数.md "wikilink")，**半环**是类似于环但没有[加法逆元的](https://zh.wikipedia.org/wiki/加法逆元 "wikilink")[代数结构](../Page/代数结构.md "wikilink")。偶尔使用术语
**rig** - 这起源于一个笑话，rig 是没有 *n*egative 元素的 ri*n*g。

## 定义

**半环**是装备了两个[二元关系](../Page/二元关系.md "wikilink") + 和 ·
的[集合](https://zh.wikipedia.org/wiki/集合 "wikilink") *R*，有着:

1.  (*R*, +) 是带有[单位元](https://zh.wikipedia.org/wiki/单位元 "wikilink") 0
    的[交换幺半群](https://zh.wikipedia.org/wiki/交换幺半群 "wikilink"):
    1.  (*a* + *b*) + *c* = *a* + (*b* + *c*)
    2.  0 + *a* = *a* + 0 = *a*
    3.  *a* + *b* = *b* + *a*
2.  (*R*, ·) 是带有单位元 1 的[幺半群](../Page/幺半群.md "wikilink"):
    1.  (*a*·*b*)·*c* = *a*·(*b*·*c*)
    2.  1·*a* = *a*·1 = *a*
3.  乘法[分配于加法之上](../Page/分配律.md "wikilink"):
    1.  *a*·(*b* + *c*) = (*a*·*b*) + (*a*·*c*)
    2.  (*a* + *b*)·*c* = (*a*·*c*) + (*b*·*c*)
4.  0 抵消 *R*:
    1.  0·*a* = *a*·0 = 0

最后的[公理可以从环的定义而省略](../Page/公理.md "wikilink"):
它可以自动的从其他环公理得出。这里不行，必须在定义中声明。

在环和半环之间的区别是加法只产生[交换幺半群](https://zh.wikipedia.org/wiki/交换幺半群 "wikilink")，而不必然是[阿贝尔群](../Page/阿贝尔群.md "wikilink")。

符号 · 经常从表示法中省略；就是说 *a*·*b* 写为
*ab*。类似的，接受一种[运算次序](https://zh.wikipedia.org/wiki/运算次序 "wikilink")，·
先于 + 应用；就是说 *a* + *bc* 就是 *a* + (*bc*)。

**交换半环**是乘法为[交换性的半环](https://zh.wikipedia.org/wiki/交换律 "wikilink")。**等幂半环**(也叫做
**dioid**)是加法是[等幂的半环](https://zh.wikipedia.org/wiki/等幂 "wikilink"): *a*
+ *a* = *a*，就是说 (*R*, +)
是[带](https://zh.wikipedia.org/wiki/带 "wikilink")。

有些作者偏好省略半环有 0 或 1
的要求。这使得在环与半环同[群与](../Page/群.md "wikilink")[半群之间的类比更像](../Page/半群.md "wikilink")。这些作者经常称这里定义的概念为
*rig*。

**参数多态**在[程序设计语言与](../Page/程序设计语言.md "wikilink")[类型论中是指声明与定义函数](../Page/类型论.md "wikilink")、复合类型、变量时不指定其具体的类型，而把这部分类型作为参数使用，使得该定义对各种具体类型都适用。参数化多态使得语言更具表达力，同时保持了完全的静态[类型安全](../Page/类型安全.md "wikilink")。
这被称为泛型函数、泛型数据类型、泛型变量，形成了泛型编程的基础。

参数多态名字来源于其发明人[克里斯托弗·斯特雷奇](../Page/克里斯托弗·斯特雷奇.md "wikilink")，与[特设多态](../Page/特设多态.md "wikilink")（ad
hoc
polymorphism）相对。特设多态是指一个多态函数有多个不同的实现，依赖于其实参而调用相应版本的函数。因此，特设多态仅支持有限数量的不同类型。

## 直谓与非直谓

### 直谓多态

直谓参数多态（predicative parametric
polymorphism）是指，类型\(\tau\)包含类型变量\(\alpha\)不能用在\(\alpha\)被实例化为一个多态类形。直谓类型理论包括[直觉类型论与](../Page/直觉类型论.md "wikilink")

### 非直谓多态

非直谓多态（Impredicative polymorphism），也称“头等多态”（first-class
polymorphism）是最强有力的参数多态形式。[非直谓是指自引用](../Page/非直谓.md "wikilink")（self-referential），[类型论中允许实例化类型](../Page/类型论.md "wikilink")\(\tau\)的变量为任何类型，包括参数化类型，如\(\tau\)自身。一个例子是[系统F在类型变量](../Page/系统F.md "wikilink")*X*下，类型\(T = \forall X. X \to X\)，其中*X*可以为*T*自身。

[类型论中](../Page/类型论.md "wikilink")，最常被研究的非直谓[有类型λ演算是基于](../Page/有类型λ演算.md "wikilink")[λ立方体](../Page/λ立方体.md "wikilink")，特别是[系统F](../Page/系统F.md "wikilink")。

## 限定的参数多态

1985年与提出类型参数允许限定（bounds）的益处。限定量化（bounded
quantification）也称作“限定多态”（bounded
polymorphism）或“约束泛型”（constrained
genericity）。许多操作要求数据类型的某些知识，但仍可以把类型参数化。例如，判断一项是否出现在列表中，需要比较项的相等。在[Standard
ML中](../Page/Standard_ML.md "wikilink")，类型参数*’’a*被限定有相等判断可用，因此具有如下类型的函数：*’’a*
× *’’a* list →
bool且*’’a*可译为任何定义了任何定义了相等判断的类形。在[Haskell中](../Page/Haskell.md "wikilink")，限定是通过要求类型属于某个[type
class](../Page/type_class.md "wikilink")，因此同样的函数在Haskell中可写为：\({\scriptstyle Eq \, \alpha \, \Rightarrow \alpha \, \rightarrow \left[\alpha \right] \rightarrow Bool}\)。大多数支持参数多态的面向对象语言可以把参数限定为给定类型的[子类型](../Page/子类型.md "wikilink")。(见[子类型多态](../Page/子类型多态.md "wikilink"))

下述Java例子中，类型参数T被限于I的子类：

``` java
class I {
}

class A <T extends I> {
    public T id(T x) {
        return x;
    }
}
```

**字符编码探测**、**字符集探测**又稱為**代码页检测**是個[启发式猜测代表文字的一系列字节的](https://zh.wikipedia.org/wiki/启发式 "wikilink")[字符编码](../Page/字符编码.md "wikilink")。其算法通常依据对字节样式的统计分析。这并不是一个万无一失的方法因为它依赖于统计数据——比如有些[Windows版本会误把ASCII编码的](https://zh.wikipedia.org/wiki/Microsoft_Windows "wikilink")"[Bush hid the facts](https://zh.wikipedia.org/wiki/Bush_hid_the_facts "wikilink")"当作中文[UTF-16LE](https://zh.wikipedia.org/wiki/UTF-16 "wikilink")。

为数不多的能可靠探测的情况之一是探测[UTF-8](../Page/UTF-8.md "wikilink")。这是因为UTF-8中有大量的无效字节序列，所以当其他编码方式使用字节中的高位bit时*极*不可能通过UTF-8有效性测试。不幸的是不完善的字符集探测程序不优先进行可靠的UTF-8测试于是把UTF-8定为其他编码。

## 其它的編碼識別方式

由於字符集探测的不可靠，所以通常是用屬性標籤去註明是用何種編碼。\[1\]

;在HTML的<code>

<head>

</head>

</code>區塊裡 例如在HTML文件中使用一個`meta` element去標示出編碼種類： 在HTML5使用

``` html5
 <meta charset="utf-8" />
```

相當於HTML4使用

``` html4strict
 <meta http-equiv="Content-Type" content="text/html;charset=utf-8" />
```

也可以在[HTTP傳輸文件時](https://zh.wikipedia.org/wiki/HTTP "wikilink")，在HTTP檔頭裡使用Content-type通知給瀏覽器知道使用何種編碼，以使瀏覽器能正確的顯示出內容。\[2\]

  - 在asp輸出頁面前使用

<!-- end list -->

``` java
 Response.Charset="utf-8"
```

相當於使用

``` java
 Response.AddHeader "Content-Type", "text/html;charset=utf-8"
```

  - 在[JSP](../Page/JSP.md "wikilink")輸出頁面前使用

<!-- end list -->

``` java
 <%@ page contentType="text/html; charset=utf-8" %>
```

  - 在[PHP](../Page/PHP.md "wikilink")輸出頁面前使用

<!-- end list -->

``` php
 header('Content-type: text/html; charset=utf-8');
```

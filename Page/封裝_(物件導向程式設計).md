在[物件導向程式設計方法中](https://zh.wikipedia.org/wiki/物件導向程式設計 "wikilink")，**封裝**（）是指，一種將[抽象性函式介面的實作細節部份包裝](../Page/抽象化_\(計算機科學\).md "wikilink")、隱藏起來的方法。同時，它也是一種防止外界呼叫端，去存取[物件內部實作細節的手段](https://zh.wikipedia.org/wiki/物件 "wikilink")，這個手段是由程式語言本身來提供的。這兩個概念有一些不同，但通常被混合使用。封裝被視為是[物件導向的四項原則之一](https://zh.wikipedia.org/wiki/物件導向 "wikilink")。

適當的封裝，可以將物件使用介面的程式實作部份隱藏起來，不讓使用者看到，同時確保使用者無法任意更改物件內部的重要資料。它可以讓程式碼更容易理解與維護，也加強了程式碼的安全性。

## 程式範例

### C\#範例

这是一段[C\#代码](../Page/C♯.md "wikilink")，演示了如何使用`private`关键字限制变量的访问：

``` csharp
namespace Encapsulation
{
    class Program
    {
        public class Account
        {
            private decimal accountBalance = 500.00m;

            public decimal CheckBalance()
            {
                return accountBalance;
            }
        }

        static void Main()
        {
            Account myAccount = new Account();
            decimal myBalance = myAccount.CheckBalance();

            /* Main方法能够通过public的“CheckBalance”方法确认账户余额，但是不能更改它 */
        }
    }
}
```

### JAVA範例

下面是Java的演示程序:

``` java

public class Employee {
    private BigDecimal salary = new BigDecimal(50000.00);

    public BigDecimal getSalary() {
        return salary;
    }

    public static void main() {
        Employee e = new Employee();
        BigDecimal sal = e.getSalary();
    }
}
```

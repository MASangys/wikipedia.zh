在[OS
X的](https://zh.wikipedia.org/wiki/OS_X "wikilink")[Cocoa](../Page/Cocoa.md "wikilink")，[NeXTSTEP和](../Page/NeXTSTEP.md "wikilink")[GNUstep编程](../Page/GNUstep.md "wikilink")[框架中](https://zh.wikipedia.org/wiki/框架 "wikilink")，**属性列表**（Property
List）文件是一种用来存储[序列化后的](../Page/序列化.md "wikilink")[对象的文件](https://zh.wikipedia.org/wiki/对象_\(计算机科学\) "wikilink")。属性列表文件的[文件扩展名为](../Page/文件扩展名.md "wikilink")`.plist`，因此通常被称为**plist**文件。

Plist文件通常用于储存用户设置，也可以用于存储[捆绑的信息](https://zh.wikipedia.org/wiki/捆绑_\(计算机\) "wikilink")，该功能在旧式的Mac
OS中是由[资源分支提供的](https://zh.wikipedia.org/wiki/资源分支 "wikilink")。

## 表达

由于Plist中存储的数据是[抽象的](https://zh.wikipedia.org/wiki/抽象 "wikilink")，其采用的[文件格式可以不止一种](https://zh.wikipedia.org/wiki/文件格式 "wikilink")。[NeXTSTEP只使用一种格式](../Page/NeXTSTEP.md "wikilink")，而稍晚的[GNUstep和](../Page/GNUstep.md "wikilink")[OS
X框架导入了不同的文件格式](https://zh.wikipedia.org/wiki/OS_X "wikilink")。

### NeXTSTEP

[NeXTSTEP系统中](../Page/NeXTSTEP.md "wikilink")，Plist被设计为[人类可读的](https://zh.wikipedia.org/wiki/人类可读 "wikilink")，并可以手工修改，故采用了类似于[编程语言的语法将数据序列化为ASCII数据](../Page/编程语言.md "wikilink")。

[字符串被表示为](../Page/字符串.md "wikilink")：

`"This is a plist string"`

[二进制数据被表示为](https://zh.wikipedia.org/wiki/二进制文件 "wikilink")：

`< `*`[文本形式的十六进制数]`*` >`

[数组被表示为](../Page/数组.md "wikilink")：

`( "1", "2", "3" )`

[字典则被表示为](../Page/关联数组.md "wikilink")：

`{`
`    "键" = "值";`
`    ...`
`}`

NeXT的plist格式的一个限制是它不能表示NSValue对象（即数，布尔值等）。

### GNUstep

GNUstep采用了NeXTSTEP的格式，并添加了少许特性。它支持NSValue对象（以普通ASCII码表示），支持NSDate（序列化成`<*DYYYY-MM-DD
HH:MM:SS timezone>`的格式）。

GNUstep也可以读写[OS
X使用的Plist格式](https://zh.wikipedia.org/wiki/OS_X "wikilink")。

### Mac OS X

Mac OS X在支持读取NeXTSTEP格式的同时，并不鼓励使用它，而是推荐使用Apple导入的两种新格式。

[Mac OS X
10.0中](../Page/Mac_OS_X_v10.0.md "wikilink")，NeXTSTEP的格式被认为是不被推荐的，并导入了一种新的[XML格式](../Page/XML.md "wikilink")，其由Apple定义的公开[DTD所规范化](https://zh.wikipedia.org/wiki/DTD "wikilink")。XML格式支持非ASCII格式，亦可存储NSValue对象。但与GNUstep的ASCII
plist格式不同，Apple的NeXTSTEP格式的plist文件不支持这两种特性。

由于XML文件在储存时不是最有空间效率的，[Mac OS X
10.2导入了一种新的格式](../Page/Mac_OS_X_v10.2.md "wikilink")，它将plist文件存储为二进制文件。从[Mac
OS X
10.4开始](https://zh.wikipedia.org/wiki/Mac_OS_X_v10.4 "wikilink")，这是偏好设置文件的默认格式。

`plutil`工具（在[Mac OS X
10.2中导入](../Page/Mac_OS_X_v10.2.md "wikilink")）可以用来检查plist的语法，或者对plist文件进行格式转换。

可以使用任何文本编辑器对XML格式的plist进行更改，同时，Apple提供了“[Property List
Editor](https://zh.wikipedia.org/wiki/Property_List_Editor "wikilink")”应用程序（作为[Apple
Developer
Tools的一部分安装](https://zh.wikipedia.org/wiki/Apple_Developer_Tools "wikilink")），它是一个树状的查看器与编辑器，并可以处理二进制格式的plist。

对于XML格式，XML标签和相关[Foundation类](https://zh.wikipedia.org/wiki/Foundation "wikilink")、[Core
Foundation类型](https://zh.wikipedia.org/wiki/Core_Foundation "wikilink")、数据储存格式的关系如下表所示：

| Foundation类  | Core Foundation类型 | XML标签                  | 储存格式                                               |
| ------------ | ----------------- | ---------------------- | -------------------------------------------------- |
| NSString     | CFString          | <string>               | [UTF-8编码的字符串](../Page/UTF-8.md "wikilink")         |
| NSNumber     | CFNumber          | <real>, <integer>      | 十进制数字符串                                            |
| NSNumber     | CFBoolean         | <true />, or <false /> | 无数据（只有标签）                                          |
| NSDate       | CFDate            | <date>                 | [ISO 8601格式的日期字符串](../Page/ISO_8601.md "wikilink") |
| NSData       | CFData            | <data>                 | [Base64编码的数据](../Page/Base64.md "wikilink")        |
| NSArray      | CFArray           | <array>                | 可以包含任意数量的子元素                                       |
| NSDictionary | CFDictionary      | <dict>                 | 交替包含<key>标签和plist元素标签                              |

`defaults`工具可以用于以[命令行形式](https://zh.wikipedia.org/wiki/命令行 "wikilink")，通过程序的偏好设置域，管理用于储存偏好设置的plist文件；而[Property
List
Editor则可以用于编辑任何plist文件](https://zh.wikipedia.org/wiki/Property_List_Editor "wikilink")（包括用于储存偏好设置的）。从[Mac
OS X
10.4开始](https://zh.wikipedia.org/wiki/Mac_OS_X_v10.4 "wikilink")，Apple提供了通过System
Events应用程序读写plist文件的[AppleScript接口](../Page/AppleScript.md "wikilink")，而从[Mac
OS X
10.5开始](https://zh.wikipedia.org/wiki/Mac_OS_X_v10.5 "wikilink")，Apple也提供了用于编辑，创建和写入plist文件的接口\[1\]。

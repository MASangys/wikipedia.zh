**手册页**（，缩写**man page**）是在[Unix或](../Page/UNIX.md "wikilink")[类Unix](../Page/类Unix系统.md "wikilink")[操作系统](../Page/操作系统.md "wikilink")在线[软件文档](../Page/软件文档.md "wikilink")的一种普遍的形式。 内容包括[计算机程序](../Page/计算机程序.md "wikilink")（包括[库和](https://zh.wikipedia.org/wiki/库 "wikilink")[系统调用](https://zh.wikipedia.org/wiki/系统调用 "wikilink")），正式的标准和惯例，甚至是抽象的概念。[用户](../Page/用户.md "wikilink")可以通过执行`man`[命令调用手册页](../Page/命令_\(计算机\).md "wikilink")。

## 用法

为查阅某个Unix命令的手册页，用户可以在[命令行界面执行如下命令](https://zh.wikipedia.org/wiki/Shell "wikilink")：

``` bash
man <command_name>
```

例如，“`man ftp`”。为了方便用户查阅输出的信息，man命令一般会提供一个。

按惯例，man页面可以按照“程序名(区段)”的标记来查阅：例如。相同的手册页标记可能出现在多个不同的区段（section）中，例如相同的名字在[系统调用](https://zh.wikipedia.org/wiki/系统调用 "wikilink")、用户[命令或](../Page/命令_\(计算机\).md "wikilink")[宏软件包中重复出现的时候](../Page/Troff.md "wikilink")。比如与 ，以及与。

访问非默认手册页区段的语法在不同的软件实现中是不一样的。例如，在[Solaris](../Page/Solaris.md "wikilink")中，阅读的语法为：

``` bash
man -s 3c printf
```

在[Linux](../Page/Linux.md "wikilink")和类BSD系统中，相同的命令语法为：

``` bash
man 3 printf
```

此命令在手册页的第3区段中搜寻“[printf](https://zh.wikipedia.org/wiki/printf "wikilink")”的信息。

## 历史

[Screenshot_of_"Xman"_program.png](https://zh.wikipedia.org/wiki/File:Screenshot_of_"Xman"_program.png "fig:Screenshot_of_\"Xman\"_program.png")应用，用于查看手册页\]\] [OpenBSD_Manpages_Section_8_Intro.png](https://zh.wikipedia.org/wiki/File:OpenBSD_Manpages_Section_8_Intro.png "fig:OpenBSD_Manpages_Section_8_Intro.png") intro第8区段的手册页\]\]

《[UNIX程序员手册](http://man.cat-v.org/unix-1st/)》1971年11月3日第一次出版。1971年，在主管[道格拉斯·麥克羅伊](../Page/道格拉斯·麥克羅伊.md "wikilink")的坚持下，[丹尼斯·里奇](../Page/丹尼斯·里奇.md "wikilink")和[肯·汤普逊](../Page/肯·汤普逊.md "wikilink")编写了第一份实际意义上的手册页。除了手册页，《UNIX程序员手册》还汇编了一些简短的论文，其中一部分是（例如Unix的一般用法、[C语言](https://zh.wikipedia.org/wiki/C语言 "wikilink")，以及[Yacc](../Page/Yacc.md "wikilink")等工具），另一部分是对操作系统功能的更详细的说明。手册的印刷版最初装在一个活页夹中，但和[Research Unix](../Page/Research_Unix.md "wikilink")[第七版的手册被分为两卷](https://zh.wikipedia.org/wiki/Version_7_Unix "wikilink")，印刷版的手册页成为了其第1卷。\[1\]

手册页是用[troff](https://zh.wikipedia.org/wiki/troff "wikilink")[排版](../Page/排版.md "wikilink")[软件包排版的](https://zh.wikipedia.org/wiki/软件包 "wikilink")，是一组`-man`[宏](https://zh.wikipedia.org/wiki/宏 "wikilink")。当时，手册页系统带来的联机文档可用性被认为是一项伟大的进步。时至今日，几乎所有的Unix命令行应用程序都附带了手册页，而很多Unix用户将缺少手册页视为低质量应用程序的一个标志。事实上，有些项目（如[Debian](../Page/Debian.md "wikilink")）不辞劳苦的为缺乏手册页的应用编写手册页；[4.4BSD的现代继任者也将手册页作为系统文档的主要形式之一](../Page/BSD.md "wikilink")（将旧的`-man`宏用新的`-mdoc`宏取代）。

几乎没有什么替代品能像`man`那样流行，只有[GNU計劃](../Page/GNU計劃.md "wikilink")的[`info`](https://zh.wikipedia.org/wiki/Texinfo "wikilink")系统\[2\]可能是个例外。 此外，一些Unix[图形用户界面](../Page/图形用户界面.md "wikilink")应用程序（尤其是那些使用[GNOME](../Page/GNOME.md "wikilink")和[KDE](../Page/KDE.md "wikilink")开发环境构建的）现在提供的[HTML](../Page/HTML.md "wikilink")版本的最终用户手册，并包含内嵌的HTML阅读器，如`Yelp`，以供阅读应用程序中的帮助文档。

手册页通常是用英文写的，在某些系统上可能有翻译成其他语言的版本。

手册页的默认格式是[troff](https://zh.wikipedia.org/wiki/troff "wikilink")，使用man（着重展现）或[mdoc宏软件包](https://zh.wikipedia.org/wiki/mdoc "wikilink")（着重语义）。可以把手册页排版成[PostScript](../Page/PostScript.md "wikilink")、[PDF和其他各种格式进行查看或打印](https://zh.wikipedia.org/wiki/PDF "wikilink")。

大多数[Unix系统有一个man](https://zh.wikipedia.org/wiki/Unix "wikilink")2html命令的[软件包](https://zh.wikipedia.org/wiki/软件包 "wikilink")，该命令允许用户使用HTML浏览器浏览手册页（例如，[FreeBSD](../Page/FreeBSD.md "wikilink")的textproc/man2html，或某些[Linux](../Page/Linux.md "wikilink")发行版中的man）。

2010年，[OpenBSD](../Page/OpenBSD.md "wikilink")宣布淘汰[troff](https://zh.wikipedia.org/wiki/troff "wikilink")，改用\[3\]展示手册页。

## 手册区段

在[Research Unix](../Page/Research_Unix.md "wikilink")、[BSD](../Page/BSD.md "wikilink")、[OS X和](https://zh.wikipedia.org/wiki/OS_X "wikilink")[Linux](../Page/Linux.md "wikilink")中，手册通常被分为8个区段，安排如下：

| 区段 | 说明                                                                                                    |
| -- | ----------------------------------------------------------------------------------------------------- |
| 1  | 一般[命令](../Page/命令_\(计算机\).md "wikilink")                                                              |
| 2  | [系统调用](https://zh.wikipedia.org/wiki/系统调用 "wikilink")                                                 |
| 3  | [库函数](../Page/函式庫.md "wikilink")，涵盖[C标准函数库](https://zh.wikipedia.org/wiki/C标准函数库 "wikilink")          |
| 4  | [特殊文件](https://zh.wikipedia.org/wiki/设备文件 "wikilink")（通常是/dev中的设备）和[驱动程序](../Page/驱动程序.md "wikilink") |
| 5  | [檔案格式](../Page/檔案格式.md "wikilink")和约定                                                                 |
| 6  | [游戏和](../Page/电子游戏.md "wikilink")[屏保](../Page/螢幕保護裝置.md "wikilink")                                   |
| 7  | 杂项                                                                                                    |
| 8  | 系统管理[命令和](../Page/命令_\(计算机\).md "wikilink")[守护进程](../Page/守护进程.md "wikilink")                         |

Unix [System V采用了类似的编号方案](https://zh.wikipedia.org/wiki/System_V "wikilink")，但顺序不同：

| 区段 | 说明                                                                                                    |
| -- | ----------------------------------------------------------------------------------------------------- |

| 2  | [系统调用](https://zh.wikipedia.org/wiki/系统调用 "wikilink")                                                 |
| 3  | [C函数库函数](https://zh.wikipedia.org/wiki/C标准函数库 "wikilink")                                             |
| 4  | [檔案格式](../Page/檔案格式.md "wikilink")和约定                                                                 |
| 5  | 杂项                                                                                                    |
| 6  | [游戏和](../Page/电子游戏.md "wikilink")[屏保](../Page/螢幕保護裝置.md "wikilink")                                   |
| 7  | [特殊文件](https://zh.wikipedia.org/wiki/设备文件 "wikilink")（通常是/dev中的设备）和[驱动程序](../Page/驱动程序.md "wikilink") |
|    |                                                                                                       |

在某些系统中还有下述的区段可用：

| 区段 | 描述                                                                                              |
| -- | ----------------------------------------------------------------------------------------------- |
| 0  | [C函数库](https://zh.wikipedia.org/wiki/C标准函数库 "wikilink")[头文件](../Page/头文件.md "wikilink")         |
| 9  | [内核](../Page/内核.md "wikilink")例程                                                                |
| n  | [Tcl](../Page/Tcl.md "wikilink")/[Tk](../Page/Tk.md "wikilink")[关键字](../Page/保留字.md "wikilink") |

一些区段利用后缀进一步细分了。例如在一些系统中，区段3C是C函数库调用，3M是数学（Math）函数库，等等。这样做的结果是区段8（系统管理命令）有时也被移动到区段1M（作为[命令区段的子区段](../Page/命令_\(计算机\).md "wikilink")）。一些子区段后缀有跨区段的一般含义：

| 子区段 | 说明                                                                  |
| --- | ------------------------------------------------------------------- |
| p   | [POSIX规范](https://zh.wikipedia.org/wiki/POSIX "wikilink")           |
| x   | [X Window系统文档](https://zh.wikipedia.org/wiki/X_Window系统 "wikilink") |

## 布局

所有的手册页遵循一个常见的布局，其为通过简单的[ASCII](../Page/ASCII.md "wikilink")文本展示而优化，而这种情况下可能没有任何形式的高亮或字体控制。一般包括以下部分内容：

  - NAME（名称）: 该命令或函数的名称，接着是一行简介。
    SYNOPSIS（概要）: 对于[命令](../Page/命令_\(计算机\).md "wikilink")，正式的描述它如何运行，以及需要什么样的命令行参数。对于函数，介绍函数所需的参数，以及哪个头文件包含该函数的定义。
    DESCRIPTION（说明）: 命令或函数功能的文本描述。
    EXAMPLES（示例）: 常用的一些示例。
    SEA ALSO（参见）: 相关命令或函数的列表。

也可能存在其他部分内容，但这些部分没有得到跨手册页的标准化。常见的例子包括：OPTIONS（选项），EXIT STATUS（退出状态），ENVIRONMENT（环境），BUGS（程序漏洞），FILES（文件），AUTHOR（作者），REPORTING BUGS（已知漏洞），HISTORY（历史）和COPYRIGHT（版权）。

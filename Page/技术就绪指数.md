**技术就绪指数**（**Technology Readiness Level**，**TRL**）也稱為**技术備便水準**，是一種衡量[技術發展](https://zh.wikipedia.org/wiki/技術 "wikilink")（包括[材料](../Page/材料.md "wikilink")、[零件](https://zh.wikipedia.org/wiki/零件 "wikilink")、[設備等](https://zh.wikipedia.org/wiki/設備 "wikilink")）成熟度的指標，為部份[美國聯邦政府](../Page/美國聯邦政府.md "wikilink")的機構及國際性公司所使用，在應用相關技術前，先衡量技術的成熟度。

一般而言，當一個新的技術被發明或概念化時，不適合立刻應用在實際的[系統](../Page/系統.md "wikilink")或子系統中。新的技術需要經過許多的[實驗](https://zh.wikipedia.org/wiki/實驗 "wikilink")、改良及實際的測試。在充份證明新的技術可行性後，才會整合到系統或子系統中。

## 定義

雖然不同的機構對技术就绪指数有不同的定義，不過这些定义通常大同小异。最常用的是[美國國防部](https://zh.wikipedia.org/wiki/美國國防部 "wikilink")（DoD）及[美國國家航空航天局](https://zh.wikipedia.org/wiki/美國國家航空航天局 "wikilink")（NASA）的定義。

### [美国国防部](../Page/美国国防部.md "wikilink")定义

| 级别                         | 描述 | 为达到此级别需要的结果 |
| -------------------------- | -- | ----------- |
| 1\. 已观察到并报道基本原理            |    |             |
| 2\. 已制定技术概念和 / 或实际应用       |    |             |
| 3\. 已分析及实验关键功能和 / 或概念的定性证明 |    |             |
| 4\. 在实验室环境验证组件和 / 或面包板     |    |             |
| 5\. 在相关环境验证组件和 / 或面包板      |    |             |
| 6\. 在相关环境验证系统或子系统或原型       |    |             |
| 7\. 在实际运用环境演练系统原型          |    |             |
| 8\. 实际系统完成并通过测试和演练         |    |             |
| 9\. 已通过实际任务证明系统可实用         |    |             |

**摩斯電碼**（）是一种時斷時續的信號，通過不同的排列顺序來表達不同的[英文字母](../Page/英文字母.md "wikilink")、[數-{}-字和](https://zh.wikipedia.org/wiki/数字 "wikilink")[标點符號](../Page/标点符号.md "wikilink")。是由美國人[艾爾菲德·維爾與](https://zh.wikipedia.org/wiki/艾爾菲德·維爾 "wikilink")[薩缪爾·摩斯在](../Page/萨缪尔·摩尔斯.md "wikilink")1836年發明。

摩斯電碼是一种早期的[數碼化通信形式](https://zh.wikipedia.org/wiki/數碼化 "wikilink")，但是它不同於現代只使用0和1兩種狀態的二進制代码，它的代碼包括五種：

1.  點（`·`）：1
2.  劃（`-`）：111
3.  字符内部停頓（點和劃之間）：0
4.  字符外部停頓（在字符之間）：000
5.  單字外部停顿（在單字之間）：0000000

## 摩尔斯电码的历史

最早的摩斯電碼是一些表示數字的[點和](https://zh.wikipedia.org/wiki/點 "wikilink")[劃](https://zh.wikipedia.org/wiki/連接號 "wikilink")([連字號](https://zh.wikipedia.org/wiki/連字號 "wikilink"))。數字對應字母，需要查找一本代碼表才能知道每個字母對應的數。用一个[電鍵可以敲擊出點](https://zh.wikipedia.org/wiki/電鍵 "wikilink")、劃以及中間的停頓。

雖然摩斯發明了[電報](../Page/电报.md "wikilink")，但他缺乏相關的專門技術。他與維爾簽定了一个協議，讓他幫自己制造更加實用的設備。維爾構想了一個方法，通過點、劃和中間的停頓，可以讓每個字符和标點符號彼此獨立地發送出去。他们達成一致，同意把这種標示不同符號的方案放到摩斯的專利中。这就是现在我们所熟知的美式摩尔斯电码，它被用来传送了世界上第一条电报。

這種代碼可以用一種音調平穩時斷時續的[無線電信号來傳送](https://zh.wikipedia.org/wiki/无线电 "wikilink")，通常被稱做「[連續波](https://zh.wikipedia.org/wiki/連續波 "wikilink")（Continuous
Wave，CW）」。它可以是電報電線的電子脈衝，也可以是一种機械或視覺的信號（比如閃光）。

一般來說，任何一種能把書面字符用可變長度的信號表示的編碼方式都可以稱为摩斯電碼。但現在這一名詞只用來特指兩種表示[英文字母和符號的摩斯電碼](../Page/英语.md "wikilink")：美式摩斯電碼被使用在有線電報通信系统；今天還在使用的國際摩斯電碼則只使用點和劃。

电报公司根据要发的信的长度收费。商业代码精心设计了五个字符组成一组的代码，做为一个单词发送。比如：BYOXO（Are you trying to
crawl out of it?）；LIOUY（Why do you not answer my question?
為什麼不回覆？）；AYYLU（Not clearly coded, repeat more clearly.
編碼不清楚，請清楚重覆）。这些五个字符的简语可以用摩尔斯电码单独发送。有些简语后来在现代的网络上常用，如CUL（See you later.
再见）。

现在仍然在[业余无线电中使用的有](../Page/业余无线电.md "wikilink")[Q简语和](../Page/Q简语.md "wikilink")[Z简语](https://zh.wikipedia.org/wiki/Z简语 "wikilink")：他们最初是为报务员之间交流通信质量、频率变更、电报编号等信息服务的；還有在手提電話的訊息鈴聲中，廠商依然會將「SMS（`···--···`）」一句的摩尔斯电码加入。一些电视台及电台也会在ID中加入摩尔斯电码元素，如[無綫新聞片头曲中的](https://zh.wikipedia.org/wiki/無綫新聞 "wikilink")「NEWS
TODAY」以及德国[ZDF新闻节目](https://zh.wikipedia.org/wiki/ZDF "wikilink")《[今天](https://zh.wikipedia.org/wiki/今天_\(德国\) "wikilink")》片头曲中的“heute”。

1838年1月8日，[Alfred
Vail公司展示了一种使用点和劃的](https://zh.wikipedia.org/wiki/阿爾弗萊德·維爾 "wikilink")[电报码](../Page/电报.md "wikilink")，这是摩尔斯电码前身。

作为一种信息编码标准，摩尔斯电码拥有其他编码方案无法超越的長久生命。摩尔斯电码在海事通讯中被作为国际标准一直使用到1999年。1997年，当[法国海军停止使用摩尔斯电码时](https://zh.wikipedia.org/wiki/法国海军 "wikilink")，发送的最后一条消息是：「所有人注意，这是我们在永远沉寂之前最后的一声呐喊！」

### 美式摩尔斯电码

作为一种实际上已经绝迹的电码，美式摩尔斯电码使用不太一样的点、劃和独特地间隔来表示数字、字符和特殊符号。这种摩尔斯电码的设计主要是针对地面电报务员通过电报电线传输的，而非通过无线电波。
这种古老的、交错的电码是为了配合电报务员接听方式而设计的。不像现在可以从扬声器或者耳机中听到电码的音调，只能从这些最早期的电报机的一个机械发生装置听到嗒嗒的声音，甚至是从发送电键接听：这种电键在不发送信号时被设置为被动模式，负责发声。
这些报务员大多是为铁路或以后的西联电传等服务。像那时的许多年轻人一样，十几岁的[爱迪生就是这样一名话务员](../Page/爱迪生.md "wikilink")。

### 现代国际摩尔斯电码

现代国际摩尔斯电码是由 Friedrich Clemens Gerke
在1848年发明的，用在[德国的](../Page/德国.md "wikilink")[漢堡和](https://zh.wikipedia.org/wiki/汉堡市 "wikilink")[庫克斯港之间的电报通信](https://zh.wikipedia.org/wiki/庫克斯港 "wikilink")。1865年之后在少量修改之后由国际电报大会在巴黎标准化，后来由[国际电信联盟统一定名为国际摩尔斯电码](../Page/国际电信联盟.md "wikilink")。

在今天，国际摩尔斯电码依然被使用着，虽然这几乎完全成为了[业余无线电爱好者的专利](../Page/业余无线电.md "wikilink")。直到2003年，[国际电信联盟管理着世界各地的摩尔斯电码熟练者取得业余无线电执照的工作](../Page/国际电信联盟.md "wikilink")。在一些国家，业余无线电的一些波段仍然只为发送摩尔斯电码信号而预留。
因为摩尔斯只依靠一个平稳的不变调的无线电信号，所以它的无线电通讯设备比起其它方式的更简单，并且它能在高噪声、低信号的环境中使用。同时，它只需要很窄的[带宽](../Page/带宽.md "wikilink")，并且还可以帮助两个母语不同、在话务通讯时会遇到巨大困难的操作者之间进行沟通。它也是[QRP中最常使用的方式](https://zh.wikipedia.org/wiki/QRP "wikilink")。

在[美国](../Page/美国.md "wikilink")，直到1991年，为了获得[联邦通信委员会](../Page/联邦通信委员会.md "wikilink")（FCC）颁发的允许使用[高频波段的业余无线电证书](https://zh.wikipedia.org/wiki/高频波段 "wikilink")，必须通过每分钟五个单词（WPM）的摩尔斯码发送和接收测试。1999年以前，达到20WPM的熟练水平才能获得最高级别的业余无线电证书（额外类）；1999年12月13日，FCC把额外类的这项要求降低到13WPM。
2003年世界无线电通信大会（WRC03，国际电信联盟主办的频率分配专门会议，两年一度）做出决定，允许各国在业余无线电执照管理中自己任选是否对摩尔斯电码进行要求。虽然在美国和[加拿大还有书面上的要求](../Page/加拿大.md "wikilink")，但在一些其他国家正准备彻底去除这个要求。
熟练的爱好者和军事报务员常常可以接收（抄报）40WPM以上速度的摩尔斯码。虽然传统发报电键仍有许多爱好者在使用，但半自动和全自动的电子电键在今天使用越来越广泛。[计算机软件也经常被用来生成和解码摩尔斯码电波信号](https://zh.wikipedia.org/wiki/计算机软件 "wikilink")。

## 时间控制和表示方法

有两种「符号」用来表示字符：点（`·`）和劃（`-`），或叫「滴」（Dit）和「答」（Dah）。点的长度决定了发报的速度，并且被当作发报时间参考。下面是时间控制的图示：

`-- --- ·-· ··· ·    /    -·-· --- -·· ·`
`M   O   R   S  E  (空格)   C    O   D  E`

这里，「`-`」表示劃，「`·`」表示点。这是上面消息的准确发报时间（`=`表示有信号，`.`代表無信號，每个为一个点的长度）：

`===.===...===.===.===...=.===.=...=.=.=...=.......===.=.===.=...===.===.===...===.=.=...=`
`   ^           ^        ^       ^             ^`
`   |           劃       点      |          单词间隔`
`点劃间隔                     字符间隔`

劃一般是三个点的长度；点劃之间的间隔是一个点的长度；字符之间的间隔是三个点的长度；单词之间的间隔是七个点的长度。

初学者往往被教导发送点劃间隔短小、短而快的字符，并且在符号和单词之间夸大间隔时间。比较起来，这种方式更加容易学会。

熟悉摩尔斯码的人之间经常像这样说话或拼写（其中，「长音 / Dah」是发「awe」的音）：

`  --     ---       ·-·      ···    · /    -·-·        ---      -··   ·`
` DahDah DahDahDah DiDahDit DiDiDit Dit, DahDiDahDit DahDahDah DahDiDi Di.`

### 特殊符号（统一符号）

這是一些有特殊意義的點劃组合。它们由二個或多個字母的摩斯電碼連成一個使用，這樣可以省去正常時把它們做為兩個字母發送所必須的中間間隔時間。

| 符號                                                                              | 代碼          | 意義 |-                    | <span style="text-decoration: overline">AAAAA</span> | `·-·-·-·-·-` | 呼叫信號，表示「我有訊息發送」。 |
| ------------------------------------------------------------------------------- | ----------- | ------------------------ | ---------------------------------------------------- | ------------ | ---------------- |
| <span style="text-decoration: overline">AAA</span>                              | `·-·-·-`    | 表示「本句完，接下一句」（同句號）。       |                                                      |              |                  |
| <span style="text-decoration: overline">HH</span>                               | `········`  | 表示「有錯，從上一字重新開始」。         |                                                      |              |                  |
| <span style="text-decoration: overline">AR</span>                               | `·-·-·`     | 表示「訊息結束」。                |                                                      |              |                  |
| <span style="text-decoration: overline">AS</span>                               | `·-···`     | 等待。                      |                                                      |              |                  |
| <span style="text-decoration: overline">TTTTT</span>                            | `-----`     | 表示「我正在接收你的訊息」。           |                                                      |              |                  |
| <span style="text-decoration: overline">K</span>                                | `-·-`       | 表示「我已準備好，請開始發送訊息」。       |                                                      |              |                  |
| <span style="text-decoration: overline">T</span>                                | `-`         | 表示「字收到了」。                |                                                      |              |                  |
| <span style="text-decoration: overline">IMI</span>                              | `··--··`    | 表示「請重覆你的電碼，我不是很明白」（同問號）。 |                                                      |              |                  |
| <span style="text-decoration: overline">R</span>                                | `·-·`       | 表示「訊息已收到」。               |                                                      |              |                  |
| <span style="text-decoration: overline">SK</span>                               | `···-·-`    | 表示終止（聯絡結束）。              |                                                      |              |                  |
| <span style="text-decoration: overline">BT</span>                               | `-···-`     | 分隔符。                     |                                                      |              |                  |
| <span style="text-decoration: overline">[SOS](../Page/SOS.md "wikilink")</span> | `···---···` | 求救訊號。                    |                                                      |              |                  |

以下並不是真正的统一符號：

| 符號                                                 | 代碼          | 意義 |-            |  | `···-·` | 我將重新發送最後一個單詞 |
| -------------------------------------------------- | ----------- | ---------------- |  | ------- | ------------ |
|                                                    | `·· ··`     | 同樣               |  |         |              |
| <span style="text-decoration: overline">OOO</span> | `---------` | 錯誤（Out Of Order） |  |         |              |

## 用摩尔斯码交谈

要使用摩尔斯码进行明白无误的交流，只需要字母就够了。为了使交流更有效率，有很多国际上通用的模式。

这是一个連續波通讯的例子，发生在电台一（s1）和电台二（s2）之间：

s1:

`CQ CQ CQ DE s1 K `
`呼叫任何人（CQ），這是（DE）s1，結束（K）。`

s2:

`s1 DE s2 K`
`呼叫s1，這是s2，結束。`
`（現在兩個電台就建立通訊連接了）`

s1:

`SK`
`再見。`

s2:

`SK`
`再見。`

作为一个例子，上面的电台之间并没有交谈什么内容，这-{只}-是演示一次联络的情形罢了。

## 求救信號SOS

傳聞1912年，著名的[鐵達尼號](https://zh.wikipedia.org/wiki/鐵達尼號 "wikilink")[郵輪首航遇險時](https://zh.wikipedia.org/wiki/郵輪 "wikilink")，曾使用當時剛通過並準備使用的新求救信號[SOS](../Page/SOS.md "wikilink")（`···---···`），鐵達尼號也因此成為世界上第一艘發出[SOS電碼的船隻](../Page/SOS.md "wikilink")。而事實上，有记录的SOS求救信號首次使用是由汽船阿拉帕霍号（S.S.
*Arapahoe*）在1909年8月11日，当时它驶至[北卡罗来纳州的](../Page/北卡罗来纳州.md "wikilink")后燃油殆尽，发出救援信号36小时后被救。\[1\]另外，SOS信號在1906年即已制訂，但英國的無線電操作員当时很少使用SOS信號，他們更喜歡老式的[CQD遇難信號](../Page/CQD.md "wikilink")。而传闻则称鐵達尼號的無線電首席官員約翰·喬治·菲利普一直在發送CQD遇難信號，直到下級無線電操作員哈羅德·布萊德建議他：「發送SOS吧，這是新的呼叫信號，這也可能是你最後的機會來發送它了！」然後菲利普才在傳統的CQD求救信號中夾雜SOS信號。因為[加州人号的电报员关了电报机下班去睡觉](https://zh.wikipedia.org/wiki/加州人号 "wikilink")，该求救信號直到第二天早上才被[加州人号收到](https://zh.wikipedia.org/wiki/加州人号 "wikilink")。

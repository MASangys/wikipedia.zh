在[物件導向程式設計中](../Page/物件導向程式設計.md "wikilink")，**方法**（；；
）指的是[類別](../Page/类_\(计算机科学\).md "wikilink")（所謂的**類別方法**、**靜態方法**或**工廠方法**）、或者是[物件](../Page/物件_\(電腦科學\).md "wikilink")（所謂的**實體方法**）兩者其中之一的一種[子程式](../Page/子程式.md "wikilink")。如同[程序化程式語言的程序](../Page/程序化程式語言.md "wikilink")，一個方法通常以一系列的[語句所組成](../Page/語句_\(程式設計\).md "wikilink")，並以之完成一個動作。其可以藉由輸入一組[參數以制訂所需的動作](../Page/參數_\(電腦科學\).md "wikilink")，且一部分的方法可能會有輸出值（所謂的返回值）。方法的目的是提供一個機制，以存取（對於讀和寫）物件或類別的[私有資料儲存區](../Page/私有資料.md "wikilink")。

## 方法的種類

如前所述，實體方法附屬於特定物件，而類別或者靜態方法（又稱作共享方法）則附屬於某個類別。在典型的實踐中，實體方法傳送一個隱藏的[參考](../Page/參考.md "wikilink")（舉例來說，[`this`](../Page/this_\(電腦科學\).md "wikilink")或者[`self或者Me`](../Page/self_\(電腦科學\).md "wikilink")）給所屬的物件（不論類別或類別實體），所以它們可以存取與之相關的資料。一個類別方法的典型例子即：在給定的類別中，其內部記錄着已建立物件的計數。

[建構子類似於方法](../Page/建構子_\(電腦科學\).md "wikilink")，不過它們是在語句區塊中[建立一個類別的實體時](../Page/物件生命期.md "wikilink")，所自動呼叫的。在[Java](../Page/Java.md "wikilink")、[C++](../Page/C++.md "wikilink")、[C\#以及](../Page/C_Sharp.md "wikilink")[PHP裡](../Page/PHP.md "wikilink")，它們和所屬物件的類別，皆有相同的名稱。在[Visual
Basic裡](../Page/Visual_Basic.md "wikilink")，建構子稱為`New`，而且在[Object
Pascal裡](../Page/Object_Pascal.md "wikilink")，建構子可以有使用者所定義的名字（不過大多稱作Create）。建構子有一部分很像方法，不過也存在著許多差異，所以通常不把其視為方法。

[解構子是一個特殊的實體方法](../Page/解構子_\(電腦科學\).md "wikilink")，它是在[解構一個類別的實體時](../Page/物件生命期.md "wikilink")，所自動呼叫的。在C++裡，它們和所屬物件的類別，皆有相同的名稱，只不過在前面多了一個[波浪號](../Page/波浪號.md "wikilink")（\~）。在[Object
Pascal裡](../Page/Object_Pascal.md "wikilink")，解構子可以有使用者所定義的名字（不過大多稱作Destroy）。

抽象方法是一個[虛設代碼的方法](../Page/虛設代碼.md "wikilink")，它沒有[實作](../Page/實作.md "wikilink")。它通常用來佔住一個位置，讓子類別或者以此為原型的物件[重載](../Page/重載_\(程式設計\).md "wikilink")，並實作出相應的抽象方法。如此一來，抽象方法有助於明確表明一部分[架構](../Page/架構.md "wikilink")。

存取方法通常是一個小型、簡單的，並提供一個用來從程式的其它部分存取物件[狀態的方法](../Page/狀態.md "wikilink")。雖然它引入了一個新的[附屬](../Page/偶合_\(電腦科學\).md "wikilink")，但這是直接存取狀態資料的首選方法，因其提供了一個[抽象層](../Page/抽象層.md "wikilink")。例如，如果銀行帳戶類別提供了`getBalance()`的存取方法，其用以查詢餘額（而不是直接存取餘額資料區域），之後該部分代碼的[修改版可以實作更複雜的查詢餘額機制](../Page/修改版.md "wikilink")（例如，取得[資料庫](../Page/資料庫.md "wikilink")），而無須改變相關的代碼。一個用來改變物件狀態的存取方法，通常稱作更新方法，有時又稱作[更動方法](../Page/更動方法.md "wikilink")。提供這些方法的物件就被認為是[可變物件](../Page/可變物件.md "wikilink")。

## 靜態（共享/類別）方法

如前所述，方法可能宣告為靜態的（在Visual Basic裡是`shared`，在Object
Pascal裡是`class`），即靜態方法活動於類別層次，而非實體層次。靜態方法不能觸及一個特定的類別實體（也就是不能觸及`this``self`,`Me`等等）以下是以C‘
撰寫的靜態成員以及其客戶端的例子：

```
 public class Example
   public static void StaticExample
      // 靜態方法的程式碼
   public void InstanceExample
    M// 此處為實體方法的程式碼
   M// 可以使用THIS
 /// 上述類別的客戶端：
 // 呼叫靜態方法（不包含實體）
 ExampleClass.StaticExample
 // 呼叫實體方法
 ExampleClass objMyExample = ExampleClass
 objMyExample.InstanceExample
```

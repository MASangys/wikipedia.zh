在[计算机科学与](../Page/计算机科学.md "wikilink")[数值分析中](https://zh.wikipedia.org/wiki/数值分析 "wikilink")，**最后一位上的单位值**或称**最小精度单位**，缩写为**ULP**，是毗邻的浮点数值之间的距离，也即浮点数在保持指数部分的时候最低有效数字为1所对应的值。ULP用于度量数值计算的精度。\[1\]

例如：圆周率位于毗邻的双精度浮点数3.1415926535897927与3.1415926535897936之间。

## 定义

设[底数为](https://zh.wikipedia.org/wiki/底数_\(对数\) "wikilink")*b*，如果*x*的指数为*E*，那么ULP(*x*)
=
[机器精度](https://zh.wikipedia.org/wiki/机器精度 "wikilink")·*b*<sup>*E*</sup>。\[2\]但在计算机与数值计算的文献中，对*ULP*，*指数*与*机器精度*也可用其它方式定义。如John
Harrison给出的定义: ULP(*x*)是两个最近的跨（*straddling*）*x*的浮点数*a*与*b*的距离，即*a* ≤ *x*
≤ *b*且*a* ≠ *b*，并且在指数无上限的情形。\[3\]\[4\]这两个定义基本等价。

被所有现代浮点硬件遵从[IEEE
754标准要求基本算术运算](../Page/IEEE_754.md "wikilink")（1985年起的加法、减法、乘法、除法、求平方根，以及2008年起的“[积和熔加运算（FMA）](https://zh.wikipedia.org/wiki/积和熔加运算 "wikilink")”）的结果近似到最近的浮点数且与数学确切结果的距离在0.5
ULP范围内。这性质意味着近似结果与数学确切结果的距离是最小的（但对于居中情形，有两个毗邻的浮点数都满足上述要求）。有美誉的数值库（numeric
library）计算基本[超越函数的误差在](https://zh.wikipedia.org/wiki/超越函数 "wikilink")0.5至大约1
ULP间。仅有少数库在计算超越函数时的误差小于0.5 ULP，这一问题相当复杂，归因于。\[5\]

## 例子

找到最接近圆周率的双精度浮点数：

``` cpp
#include <cmath>
#include <cstdio>
#define PI 3.14159265358979323846264338327950288419716939937510582097494459230

int main()
{
    printf("%.17g (%a)\n", PI, PI);
    printf("%.17g (%a)\n", std::nextafter(PI, 0.0), std::nextafter(PI, 0.0));
    printf("%.17g (%a)\n", std::nextafter(PI, 4.0), std::nextafter(PI, 4.0));
}
/* Output:
3.1415926535897931 (0x1.921fb54442d18p+1)
3.1415926535897927 (0x1.921fb54442d17p+1)
3.1415926535897936 (0x1.921fb54442d19p+1)
*/
```

设*x*用[IEEE
754表示时四舍五入且舍入到偶数的操作记为RN](../Page/IEEE_754.md "wikilink")。如果ULP(*x*)的值小于等于1，那么；否则，或，取决于最低有效位（least
significant
digit）与x的指数部分。下面的[Python程序展示了这个例子](../Page/Python.md "wikilink")：

``` pycon
>>> x = 1.0
>>> p = 0
>>> while x != x + 1:
...   x = x * 2
...   p = p + 1
...
>>> x
9007199254740992.0
>>> p
53
>>> x + 2 + 1
9007199254740996.0
```

这个例子中，从开始，反复地翻倍直至。结果是2<sup>53</sup>，因为[双精度浮点数格式使用了](https://zh.wikipedia.org/wiki/双精度 "wikilink")53位有效数字，从而ulp(2<sup>53</sup>)=2<sup>53</sup>\*2<sup>-52</sup>=2
\> 1 。即，在这里双精度浮点数可表示的最小精度间隔已经是2了。

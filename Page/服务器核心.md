**服务器核心** 是一个全新的 [Microsoft](../Page/微软.md "wikilink") [Windows
Server](../Page/Windows_Server.md "wikilink") 安装选项， 在 [Windows Server
2008](../Page/Windows_Server_2008.md "wikilink")
上首次推出。服务器核心提供了一个功能简化到核心服务的服务器环境。由于其功能有限，它减少了服务和管理要求、攻击面、磁盘占用和内存使用率。Windows
Server 团队的项目经理Andrew Mason表示，开发服务器核心的一个主要的目的是为了降低Windows
Server操作系统的攻击面，并且在未来的五年内约有70%的漏洞不会影响到服务器核心版本的Windows
Server。\[1\] 值得注意的是，该版本没有[Windows资源管理器](../Page/檔案總管.md "wikilink") 。
所有的服务器配置和维护是通过 [命令提示符，或者通过远程连接使用](../Page/命令行界面.md "wikilink")
[微软管理控制台](../Page/微软管理控制台.md "wikilink") (MMC)、远程服务器管理工具和
[PowerShell来管理](../Page/Windows_PowerShell.md "wikilink")。

## 发展历史

### Windows Server 2008

Windows Server 2008 是第一个拥有服务器核心选项的Windows服务器。(所有版本,不包括[IA -
64](https://zh.wikipedia.org/wiki/IA-64 "wikilink"))。尽管没有Windows资源管理器，但是记事本和一些控制面板程序,如区域设置都是可用的。Windows
Server 2008 服务器核心不包括：[.NET 框架](../Page/.NET框架.md "wikilink"),Internet
Explorer,Windows PowerShell和其他不支持的功能。服务器核心可以配置几个基本角色:Active
Directory域服务、活动目录应用程序模式(ADAM)、DNS服务器、DHCP服务器、文件服务器、打印服务器、Windows媒体服务器、[IIS
7
服务器和Hyper](../Page/網際網路資訊服務.md "wikilink")-V服务器。服务器核心也可以用来创建一个集群以使用故障转移集群、网络负载均衡。

对于Windows Server 2008来讲，服务器核心只是一个安装选项，它具有和GUI版本相同的文件和配置。在Windows Server
2008 和2008 R2 中，如果服务器安装的是服务器核心，它不能被转换成全GUI版本，反之亦然。
管理一个服务器核心的服务器对于管理员来讲会更轻松，一个名叫“scregedit.wsf”的Windows脚本包含了许多基本设置，比如：
[自动更新](../Page/Windows_Update.md "wikilink")
的开启或关闭、开启或关闭远程桌面以及改变网页的相关设置。\[8\]

### Windows Server 2008 R2

在 [Windows Server 2008
R2中](../Page/Windows_Server_2008_R2.md "wikilink")，服务器核心包含了.NET
框架，一些应用程序(包括使用ASP.NET编写的网站和Windows PowerShell
2.0)可以在此版本下使用。因为有了.NET框架的支持，它也是第一个支持安装 [Microsoft SQL
Server](https://zh.wikipedia.org/wiki/Microsoft_SQL_Server "wikilink")
的服务器核心(一个解决方案表示只可用SQL Server 2012+).\[9\]\[10\] 服务器可使用远程管理。
新的服务器角色现在可以在服务器核心上运行，包括主动目录证书服务和文件服务角色的资源管理器组件。
[WoW64](../Page/WoW64.md "wikilink") 现在可被禁止。\[11\]

除了
DISM<font style="background-color: rgb(189, 212, 255);"></font><font color="#0b0617"></font>
命令以外，一个新的命令行服务器的配置工具(Sconfig.cmd)已经可用于配置和管理的几种常见方面的核心服务器的安装。\[12\]
现在，服务器管理员可用该配置工具<font style="background-color: rgb(254, 246, 231);">管理安装了服务器核心的服务器。</font>

### Windows Server 2012 

相较于的先前版本， [Windows Server
2012](../Page/Windows_Server_2012.md "wikilink")
可以在服务器桌面环境和服务器核心版本之间互相转换而无须重新安装。服务器核心现在是默认的安装选项。
还有一个新的安装选项，"最少的服务器界面"，允许一些GUI元素，例如MMC和服务器管理器的运行，但没有正常的桌面和默认程序，例如
[文件管理](../Page/檔案總管.md "wikilink")。\[13\]
"最少服务器界面"实际上是一个服务器角色，全GUI版本中包含了所有的角色。
一些新的角色都可以使用，像
[Windows服务器更新服务](https://zh.wikipedia.org/wiki/Windows服务器更新服务 "wikilink")
(WSUS)、[活动目录权限管理服务 以及](../Page/Active_Directory.md "wikilink")
[路由和远程访问服务](../Page/路由和远程访问服务.md "wikilink")，并且支持许多了新特性。\[14\]

### Windows Server 2012 R2

在 [Windows Server 2012 R2](../Page/Windows_Server_2012.md "wikilink") 中,
[Windows Defender](../Page/Windows_Defender.md "wikilink")
已经在服务器核心安装中可用。在一般情况下，Windows Defender
被默认安装并启用。\[15\]

### Windows Server 2016

[Windows Server 2016](../Page/Windows_Server_2016.md "wikilink")
有一个最小状态安装选项(小于核心服务器)，它被称作微型服务器，用于优化Windows Server
虚拟化和Hyper-V 虚拟化，以及其他 云端虚拟化 方案。 微软删除了GUI 层，WoW64和 [Windows
Installer](../Page/Windows_Installer.md "wikilink")。 该版本也没有本地登录或
[远程桌面连接](https://zh.wikipedia.org/wiki/远程桌面连接 "wikilink")
的GUI支持。\[16\]\[17\]

在安装 Windows Server 2016时提供两种选择：Windows Server 2016(服务器核心)，Windows Server
2016（桌面体验）。从Windows Server 2012开始，服务器核心和桌面体验之间的转换被移除。(该特性可能在技术预览第三版被移除)

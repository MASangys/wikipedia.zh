**比特垃圾桶**（）或**比特桶**是[计算机行话](../Page/计算机行话.md "wikilink")，指丢失的计算机数据所在的地方。如果数据因传输错误或者系统崩溃丢失，或者其他在任何意义上以任何形式而丢失，就可以说数据被丢进了比特垃圾桶，一个存储着所有丢失地计算机数据的神秘地点，例如：

[Univac_bit_bucket.JPG](https://zh.wikipedia.org/wiki/File:Univac_bit_bucket.JPG "fig:Univac_bit_bucket.JPG")计算机[打孔卡片的碎纸](../Page/打孔卡片.md "wikilink")。\]\]

比特垃圾桶的原始含义是一个电传打字机或者IBM打孔卡片机器上的盒子，用来收集来自纸带或者打孔卡片的纸屑\[1\]。随后变为了一个改变为形容一切无用数据所去地点的术语，这就是[空设备](../Page/空设备.md "wikilink")，一个计算机中很有用的概念。\[2\]
有时，这样的设备也戏仿八十年代[一次写入多次读取](../Page/一次写入多次读取.md "wikilink")（英语：Write Once
Read
Many，缩写：WORM）的[磁光碟而被称为](../Page/磁光碟.md "wikilink")“一次写入永不读取”（英语：Write
Once Read
Never，缩写：WORN）设备。在Signetics公司1972年一篇玩笑数据手册中，一次写入永不读取设备随[先进不出队列和](../Page/先进不出.md "wikilink")[只写存储器而一同被提出](../Page/只写存储器.md "wikilink")。在1988年[愚人节出版的](../Page/愚人节.md "wikilink")《Compute\!》杂志中，[雅达利](../Page/雅达利.md "wikilink")[BASIC语言的作者Bill](../Page/BASIC语言.md "wikilink")
Wilknson则披露，雅达利在Atari
800系列计算机的操作系统中，作为[程序彩蛋实现了一次写入永不读取设备](../Page/彩蛋_\(媒体\).md "wikilink")。\[3\]

在编程语言中，这个术语则指一种不会被诸如[中央处理器或](../Page/中央处理器.md "wikilink")[内存使用](../Page/内存.md "wikilink")，而丢弃向其写入的一些数据的比特流。例如，在[.NET框架中是](../Page/.NET框架.md "wikilink")，则是*System.IO.Stream.Null*。\[4\]

## 编程语言

[C++11](../Page/C++11.md "wikilink")
中提供了`std::ignore`，其`operator=`是空操作、即丢弃对其所做的任何赋值，作为占位符搭配`std::tie`使用。例：

``` c++
#include <tuple>

struct S{
public:
  bool b;
  int i;
  std::string str;
};
S fn(void){...};

bool b;
int i;
std::tie(b, i, std::ignore)=fn();
```

如果不这样，那么需要用一个对象来接住返回值：

``` c++
S sRet=fn();
b=sRet.b;
i=sRet.i;
```

多了两行代码。

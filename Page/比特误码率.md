在[数据传输中](../Page/数据传输.md "wikilink")，**比特差错**（）的数量就是接收到的[信道中](../Page/信道.md "wikilink")由于[噪声](../Page/雜訊.md "wikilink")、[干扰](https://zh.wikipedia.org/wiki/interference_\(communication\) "wikilink")、[失真或](../Page/失真.md "wikilink")错误而更改的[比特的数量](../Page/位元.md "wikilink")。

**误比特率**（）是指单位时间差错比特的数量。**比特差错率**（即**误码率**，）是一段时间内差错比特的数量除以传输的总比特数。BER是一种无单位的性能指标，通常以[百分比的形式表示](../Page/百分比.md "wikilink")。\[1\]

**比特差错概率**（即**误码概率**，）*p<sub>e</sub>*
是误码率的[期望值](../Page/期望值.md "wikilink")。误码率可以视作误码概率的约略估计。对于长时间段和高差错比特，这个估计比较准确。

## 例子

举一个例子，假设传输的比特序列为：

0 1 1 0 0 0 1 0 1 1

而接收到的比特序列为：

0 <u>0</u> 1 0 <u>1</u> 0 1 0 <u>0</u> 1,

在本例中，差错比特（加下划线的比特）的数量为3。误码率为差错比特数3除以传输的比特数10，也就是0.3或者30%。

## 包错误率

**包错误率**（**误包率**，）是错误接收的[数据包的数量除以接收到的包的总数](https://zh.wikipedia.org/wiki/数据包 "wikilink")。若包中至少有一个比特是错误的，则会被认为是差错的。PER的期望值称为**包错误概率**
*p<sub>p</sub>*。设差错比特间相互独立，长度为 *N* 比特的数据包的PER就可以表示为

\[p_p = 1 - (1 - p_e)^N\],

对于很小的误码概率约为

\[p_p \approx p_eN.\]

类似指标也可以用于衡量[帧](../Page/帧.md "wikilink")、[块或](../Page/块_\(数据存储\).md "wikilink")[符号的传输](../Page/符号.md "wikilink")。

## 影响误码率的因素

在通信系统中，接收端误码率会受到传输信道[噪声](../Page/雜訊.md "wikilink")、[干扰](https://zh.wikipedia.org/wiki/串扰 "wikilink")、[失真](../Page/失真.md "wikilink")、问题、、无线[多径](https://zh.wikipedia.org/wiki/Multipath_propagation "wikilink")[衰落等的影响](https://zh.wikipedia.org/wiki/衰落 "wikilink")。

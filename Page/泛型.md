**泛型程序设计**（generic
programming）是[程序设计语言的一种风格或](../Page/程序设计语言.md "wikilink")[范式](../Page/编程范型.md "wikilink")。泛型允许程序员在[强类型程序设计语言中编写代码时使用一些以后才指定的](../Page/强类型程序设计语言.md "wikilink")[类型](../Page/类型.md "wikilink")，在[实例化时作为参数指明这些类型](../Page/实例.md "wikilink")。各种程序设计语言和其编译器、运行环境对泛型的支持均不一样。[Ada](../Page/Ada.md "wikilink")、[Delphi](../Page/Delphi.md "wikilink")、[Eiffel](../Page/Eiffel.md "wikilink")、[Java](../Page/Java.md "wikilink")、[C\#](../Page/C♯.md "wikilink")、[F\#](../Page/F#.md "wikilink")、[Swift](../Page/Swift_\(程式語言\).md "wikilink")
和 [Visual Basic .NET](../Page/Visual_Basic_.NET.md "wikilink")
称之为泛型（generics）；[ML](../Page/ML语言.md "wikilink")、[Scala](../Page/Scala.md "wikilink")
和 [Haskell](../Page/Haskell.md "wikilink")
称之为[参数多态](../Page/参数多态.md "wikilink")（parametric
polymorphism）；[C++](../Page/C++.md "wikilink") 和
[D称之为](../Page/D語言.md "wikilink")[模板](../Page/模板_\(C++\).md "wikilink")。具有广泛影响的1994年版的《Design
Patterns》一书称之为参数化类型（parameterized type）。

## 泛型的定義及目的

泛型的定義主要有以下兩種：

1.  在程序编码中一些包含**类型参数**的类型，也就是说泛型的参数只可以代表-{zh-hans:类;
    zh-hant:类别;}-，不能代表个别-{zh-hans:对象;
    zh-hant:物件;}-。（这是当今较常见的定义）
2.  在程序編碼中一些包含參數的[-{zh-hans:类;
    zh-hant:類別;}-](../Page/类_\(计算机科学\).md "wikilink")。其參數可以代表-{zh-hans:类;
    zh-hant:類別;}-或-{zh-hans:对象;
    zh-hant:物件;}-等等。（現在人們大多把這稱作[模板](../Page/模板.md "wikilink")）

不論使用哪個定義，泛型的參數在真正使用泛型時都必須作出指明。

一些[强类型程序语言支持泛型](../Page/强类型.md "wikilink")，其主要目的是加强[类型安全及减少](../Page/类型安全.md "wikilink")-{zh-hans:类;
zh-hant:类别;}-转换的次数，但一些支持泛型的程序语言只能达到部份目的。

## 偽代碼例子

``` text
類　例泛類<T> {
　　值 : T

　　設置值(新值 : T) {
　　　　值 := 新值
　　}

　　獲取值() : T {
　　　　返回　值
　　}
}

例方法1() {
　　例物件 : 例泛類<整數型>
　　例物件 := 新　例泛類<整數型>()
　　例物件.設置值(5)
　　输出整数(例对象.获取值())
}

例方法2() {
　　例物件 : 例泛類<浮點數型>
　　例物件 := 新　例泛類<浮點數型>()
　　例物件.設置值(5.5)
　　输出浮点数(例对象.获取值())
}
```

在這例子中，`例泛-{A|zh:類;zh-hans:类;zh-hant:類;zh-cn:类;zh-tw:類;zh-hk:類;zh-sg:类;zh-mo:類;}-`是一個泛型，而`T`是一個類型參數。在`例泛-{A|zh:類;zh-hans:类;zh-hant:類;zh-cn:类;zh-tw:類;zh-hk:類;zh-sg:类;zh-mo:類;}-`中沒指明`T`的實際類型，只有`例方法1()`和`例方法2()`在使用`例泛-{A|zh:類;zh-hans:类;zh-hant:類;zh-cn:类;zh-tw:類;zh-hk:類;zh-sg:类;zh-mo:類;}-`時才加以指明。

運行這例子的`例方法1()`將輸出整數5，而運行`例方法2()`將輸出浮點數5.5。

## 一些程序语言的泛型特性

### .NET 的泛型

[.NET](../Page/.NET框架.md "wikilink") 泛型的参数只可以代表-{zh-hans:类;
zh-hant:类别;}-，不能代表个别-{zh-hans:对象; zh-hant:物件;}-。由于 .NET
泛型的类型参数之实际类型在运行时均不会被消除，运行速度会因为类型转换的次数减少而加快。另外，使用`GetType()`方法可于程序运行时得知泛型及其类型参数的实际类型，更可以运用[反射编程](../Page/反射_\(计算机科学\).md "wikilink")。

.NET 允許對個別泛型的類型參數進行約束，包括以下幾種形式\[1\]（假設`T`是泛型的類型參數，`C`是一般-{zh-hans:类;
zh-hant:類別;}-、泛類，或是泛型的類型參數）：

  - `T`是一個-{zh-hans:类; zh-hant:類別;}-。
  - `T`是一個值類型。
  - `T`具有無參數的公有建構方法。
  - `T`实现-{zh-hans:接口; zh-hant:界面;}-`I`。
  - `T`是`C`，或繼承自`C`。

### Java 的泛型

Java 泛型的参数只可以代表-{zh-hans:类; zh-hant:类别;}-，不能代表个别-{zh-hans:对象;
zh-hant:物件;}-。由于Java泛型的类型参数之实际类型在编译时会被消除，所以无法在运行时得知其类型参数的类型，而且无法直接使用基本值类型作为泛型类型参数。Java编译程序在编译泛型时会自动加入类型转换的编码，故运行速度不会因为使用泛型而加快。

由于运行时会消除泛型的对象实例类型信息等缺陷经常被人詬病，Java及JVM的开发方面也尝试解决这个问题，例如Java通过在生成字节码时添加类型推导辅助信息，从而可以通过反射接口获得部分泛型信息。通过改进泛型在JVM的实现，使其支持基本值类型泛型和直接获得泛型信息等。

Java允許對個別泛型的類型參數進行約束，包括以下兩種形式\[2\]（假設`T`是泛型的類型參數，`C`是一般-{zh-hans:类;
zh-hant:類別;}-、泛類，或是泛型的類型參數）：

  - `T`实现接口`I`。
  - `T`是`C`，或繼承自`C`。

### C++的泛型（模板）

[C++](../Page/C++.md "wikilink") 泛型的参数可以代表-{zh-hans:类;
zh-hant:类别;}-或个别-{zh-hans:对象; zh-hant:物件;}-。在一般意义上，C++
缺乏对泛型的类型参数进行直接约束的手段，但可利用
SFINAE（模板代换失败非错误，指在模板实例化过程中的错误仅意味此次代换失败，并不一定产生编译错误）规则及
[C++11](../Page/C++11.md "wikilink") 的 static_assert 等实现相似功能。

``` c++
#include <type_traits>

class B{
...
};
class D: public B{
...
};
template<typename T>
void SFINAE(const std::enable_if_t<std::is_base_of<B, T>::value, T> &t);
template<typename T>
void STATIC_ASSERT(const T &t){
    static_assert(std::is_pod<T>::value, "Use with POD types only!");
}
```

如上所示，std::enable_if（std::enable_if_t\<boolean, Type\> 是
std::enable_if\<boolean, Type\>::type 的缩写）利用 SFINAE
规则来实现模板类型参数约束的手段之一。其实现方式是若布尔判断为假，则把类型设为
void，而这将导致 const void & 这种不合法的类型出现，从而禁止这种类型参数的使用。

static_assert 则在布尔判断失败时把后面的字符串作为消息内容报告为编译错误。

在编译时，每个被使用的**封闭泛型类型**（即是所有泛型参数的实际类型都已被指明的泛型）都会有独立的编码产生，编译程序会在此时确保类型安全性。可是如果泛型要运用其泛型参数的某成员，而该泛型参数又不包含该成员的时候，编译程序所产生的错误信息或会看似与实际问题无关，增加除错的难度。

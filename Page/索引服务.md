**索引服务**（**Indexing Service**，旧称**Index
Server**）是一个维护文件索引的[Windows服务](../Page/Windows服务.md "wikilink")，它可以维护计算机上的大部分[文件以改善搜索计算机和企业](../Page/電腦檔案.md "wikilink")[计算机网络的性能](../Page/计算机网络.md "wikilink")，并且更新索引无需用户介入。在[Windows
7中](../Page/Windows_7.md "wikilink")，它被新的[Windows搜索索引器取代](../Page/Windows搜索.md "wikilink")。另外，有插件可以扩展索引能力以支持更多文件格式，并且协议兼容旧版索引服务与新版[Windows搜索的索引器](../Page/Windows搜索.md "wikilink")。

## 历史

索引服务是作为一个[桌面搜索服务在](../Page/桌面搜索.md "wikilink")[Windows NT 4.0 Option
Pack](../Page/Windows_NT_4.0.md "wikilink")\[1\]和[Windows
2000及之后的版本中提供](../Page/Windows_2000.md "wikilink")。\<ref
name="MIS-v3" /\<ref name="TnC-144"
/\[2\]，那时它作为微软服务器软件[IIS的内容搜索系统](../Page/IIS.md "wikilink")。它的起源最早可追溯到微软的Cairo操作系统项目，那时它作为[对象文件系统的内容索引器](../Page/WinFS.md "wikilink")。Cairo项目最后被放弃，但内容索引功能成为了之后的Windows桌面和服务器操作系统的组件。例如在[Windows
2000中](../Page/Windows_2000.md "wikilink")，其搭载了索引服务3.0。

在[Windows
Vista中](../Page/Windows_Vista.md "wikilink")，内容索引服务被默认启用的[Windows搜索索引器取代](../Page/Windows搜索.md "wikilink")。索引服务在Windows
Server 2008中仍然存在，但不会默认安装与运行。\[3\]

索引服务在Windows 7和Windows Server 2008 R2中就已被弃用。\[4\]并在[Windows
8中被移除](../Page/Windows_8.md "wikilink")。

## 搜索界面

在初步建立索引后可进行全面的搜索。建立索引可能花费几小时乃至几天，这取决于特定目录的大小、硬盘速度、用户活动、索引器设置和其他因素。使用索引服务搜索时也可使用UNC路径和映射的网络驱动器，如果共享服务器索引了适当的目录并且意识到正在共享。

在索引服务被打开并建立适当的索引后，可以多个方面用它搜索：[Windows任务栏的](../Page/Microsoft_Windows.md "wikilink")[开始菜单在索引服务可用时会使用它搜索](../Page/开始菜单.md "wikilink")，甚至接受复杂的查询。在[微软管理控制台的](../Page/微软管理控制台.md "wikilink")“计算机管理”扩展的“服务查询表单”中可以用它执行查询。以及，第三方应用程序（例如“Aim
at File”和“Grokker Desktop”）可以使用它。

Microsoft Index Server 2.0不检测处在挂载分区中的目录的变更。由于文件系统的技术原因，它不支持挂载的卷。\[5\]

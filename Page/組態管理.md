**組態管理**（，簡稱
CM），或譯為**配置管理**，是一個建立[系統工程的過程](https://zh.wikipedia.org/wiki/系統工程 "wikilink")，用來建立與維持一個產品，讓這個產品的效能、功能，以及它所要求的物理特性，在它的生命週期中，都能保持穩定與一致性。

在[資訊與](https://zh.wikipedia.org/wiki/資訊 "wikilink")[電信的領域](https://zh.wikipedia.org/wiki/電信 "wikilink")，組態管理、或組態控制（configuration
control）擁有以下的意涵:

1.  從一特定[系統壽命的開發期到營運期](../Page/系統.md "wikilink")，憑藉針對[自動化資料系統的](https://zh.wikipedia.org/wiki/自動化資料系統 "wikilink")[硬體](https://zh.wikipedia.org/wiki/硬體 "wikilink")、[軟體](https://zh.wikipedia.org/wiki/軟體 "wikilink")、[韌體](../Page/韌體.md "wikilink")、[文件](https://zh.wikipedia.org/wiki/文件 "wikilink")、[測試](https://zh.wikipedia.org/wiki/測試 "wikilink")、[測試設備](https://zh.wikipedia.org/wiki/測試設備 "wikilink")、以及[測試文件所做的變更的管制](https://zh.wikipedia.org/wiki/測試文件 "wikilink")，進行安全特點與安全保證的管理，[原始碼管理](https://zh.wikipedia.org/wiki/原始碼 "wikilink")、或[版本控制是其中的一部份](../Page/版本控制.md "wikilink")；
2.  系統[生命週期內從頭到尾](https://zh.wikipedia.org/wiki/生命週期 "wikilink")，針對硬體、軟體、韌體、以及文件所做的變更（包括其衍生出的紀錄）的管制；
3.  針對[複雜系統的發展的管制](https://zh.wikipedia.org/wiki/複雜系統 "wikilink")、與適應。是一種規則，能讓逐步成形的軟體產品持續處於控制之下，因此能夠滿足品質，延緩限制。[軟體形態管理](../Page/软件配置管理.md "wikilink")（或簡稱[SCM](https://zh.wikipedia.org/wiki/SCM "wikilink")）可劃分為二個區域，SCM
    的第一個（較為早期的）區域，是關於在軟體開發方案期間所產生的實體的儲存，有時會交付[構件倉庫管理](https://zh.wikipedia.org/wiki/構件倉庫管理 "wikilink")（Component
    Repository
    Management，簡稱[CRM](https://zh.wikipedia.org/wiki/CRM "wikilink")）。SCM
    的第二個區域，是關於這些實體的生產、亦或變更的活動，[工程支援一詞也經常關聯於這第二個區域](https://zh.wikipedia.org/wiki/工程支援 "wikilink")；
4.  當建立了一個（電信、或資訊系統的）[形態之後](https://zh.wikipedia.org/wiki/形態 "wikilink")，針對該形態、或與其相關聯的系統構件所做的評估、與變更核准；
5.  針對[分散佇列雙匯流排](https://zh.wikipedia.org/wiki/分散佇列雙匯流排 "wikilink")（Distributed
    Queue Dual
    Bus，簡稱DQDB）型式的網路，保證一特定DQDB網路的所有節點上的資源，都會被安裝到一個正確的[雙匯流排拓蹼](https://zh.wikipedia.org/wiki/雙匯流排拓蹼 "wikilink")（dual-bus
    topology）的功能。這些被控管的功能，包括匯流排的表頭資料（Head of
    Bus，簡稱[HOB](https://zh.wikipedia.org/wiki/HOB "wikilink")）、外部計時來源、以及內定的[訊槽產生器](https://zh.wikipedia.org/wiki/訊槽產生器 "wikilink")（Slot
    generator）等功能；

## 軟體組態管理（SCM）的產品

更完整的列表參見[版本控制軟體列表](https://zh.wikipedia.org/wiki/版本控制軟體列表 "wikilink")。

  - Cfengine, administers and configures computers according to
    high-level language configuration files
  - [CodeBeamer](http://www.intland.com),a web-based collaborative
    development tool including tracker with workflow, CMDB, document
    management, forum, wiki, source code browsing functions, and
    Escalation Management（Optional）, supports Eclipse, ANT, CVS,
    Subversion, Git, Mercurial, etc.
  - KONFIG CM,
    \[[http://www.auto-trol.com/\]the](http://www.auto-trol.com/%5Dthe)
    total configuration management solution for software, hardware, data
    and all physical items.
  - LCFG, a large scale UNIX configuration system
  - [Puppet](http://reductivelabs.com/projects/puppet), administers and
    configures computers according to high-level configuration files
  - [ControlTier](http://open.controltier.com/)open source configuration
    management and process automation tools for deploying multi-tier
    enterprise applications.
  - [NetDirector](http://www.netdirector.org/)NetDirector is an open
    source configuration management tool for managing open source
    applications running on many variants of Linux. Unique features
    include the ability to simultaneously push changes out to many
    different platforms.
  - [IsiSetup](http://www.isisetup.ch/) provides a commandline interface
    to manage your system configuration files using a revision control
    tool. It runs on most Debian-based distributions and integrates well
    into the Debian package management system.
  - [Changelogic](http://www.changelogic.com/) enables web based branch,
    task and release management. It supports CVS and Subversion for
    underlying version control.
  - [Configuration Management Kit](http://www.configurationkit.com/) a
    collection of templates, draft plans and similar resources to
    support CM implementation..
  - [SMS 2003 Desired Configuration
    Monitoring](http://www.microsoft.com/technet/itsolutions/cits/mo/sman/dcm.mspx)
    a free download to enable definition of configuration items,
    automated configuration compliance auditing and compliance reporting
  - [Mercury Universal
    CMDB](http://www.mercury.com/us/products/business-availability-center/universal-cmdb),
    a toolset consisting of a business/service data model with built-in
    auto-discovery of configurations dependencies, visualization and
    business services mapping, and capabilities to track and measure
    configuration changes.

  - [ZENmetrics](https://web.archive.org/web/20080517043215/http://413technologies.co.uk/),

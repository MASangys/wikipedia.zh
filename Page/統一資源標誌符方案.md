在[计算机科学](../Page/计算机科学.md "wikilink")，
**[URI](https://zh.wikipedia.org/wiki/統一資源標誌符 "wikilink")**方案是*統一資源標誌符*
(URI)命名結構的第一級。所有的[统一资源定位符是從模式名稱後跟](../Page/统一资源定位符.md "wikilink")[兩個點構建的](../Page/冒号.md "wikilink")（“:”）構建的。對[句法和](../Page/语法学.md "wikilink")[語義的具體一部分](../Page/语义学.md "wikilink")，在很大程度上，留給不同的。

由於大多數模式最初設計用於[協議](../Page/网络传输协议.md "wikilink")，因此它們通常被錯誤地稱為“協議”或“URI協議”。例如，[超文本傳輸協定方案通常用於使用超文本傳輸協定協議指定網絡資源](https://zh.wikipedia.org/wiki/超文本傳輸協定 "wikilink")。現在，此模式中的URI用於其他應用程序，例如與協議無關的[資源描述框架资源](../Page/資源描述框架.md "wikilink")（请参阅）或XML的命名空間。

## 通用的语法

`<方案名稱>:<等級部分>[吗？ <查询>][#<片段>]`

### 例子

以下是URI的兩個示例及其不同的組件：

`  foo://username:password@example.com:8042/over/there/index.dtb?type=animal;name=ferret#nose`
`  \ /   ________________/_________/ __/            ___/ _/ _____________________/ __/`
`   |           |               |       |                |    |           |                |`
`   |          用戶            主機    端口              |    |          查詢             片段`
`   |    _______________________________/ _____________|____|/`
`  方案                  |                         |     |    |`
`   |                   域名                      路徑   |    |`
`   |                                                    |    |`
`   |             路徑                              可以解釋為文件名`
`   |   ___________|____________                              |`
`  / \ /                        \                             |`
`  `<urn:example:animal:ferrett:nose>`                  可以解釋為擴展名`

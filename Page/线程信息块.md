**Win32线程信息块**（TIB）是32位[Windows操作系统的](https://zh.wikipedia.org/wiki/Windows操作系统 "wikilink")[线程使用的](../Page/线程.md "wikilink")[数据结构](../Page/数据结构.md "wikilink")，存储了每个线程的[运行时信息](https://zh.wikipedia.org/wiki/运行时 "wikilink")。
也称作“线程环境块”（Thread Environment Block，TEB）。\[1\]

[Windows NT系列的DDK在winnt](../Page/Windows_NT.md "wikilink").h中包括了一个struct
NT_TIB，为独立于subsystem的部分。[Wine包含了TIB与subsystem相关的扩展部分](../Page/Wine.md "wikilink")。由于非常多程序使用了TIB，事实上TIB成为[Windows
API的一部分](../Page/Windows_API.md "wikilink")。\[2\]

TIB可用于获取很多进程相关信息，而不必调用Win32
API。例如，模拟GetLastError()或GetVersion()。通过[PEB可以获取访问导入表](https://zh.wikipedia.org/wiki/进程环境块 "wikilink")（import
table， IAT）、进程启动参数、程序名字等。

32位程序通过FS段寄存器，64位程序通过GS段寄存器可以获得TIB

## 访问TIB

``` c
// gcc (AT&T-style inline assembly).
void *getTIB() {
    void *pTIB;
    __asm__("movl %%fs:0x18, %0" : "=r" (pTIB) : : );
    return pTIB;
}
```

``` c
// Microsoft C
__declspec(naked)
void *getTIB() {
    __asm mov EAX, FS:[18h]
}
```

``` c
// Using Microsoft's intrinsics instead of inline assembly (works for both X86 and X64 architectures)
void *getTIB() {
#ifdef _M_IX86
    return (void *)__readfsdword(0x18);
#elif _M_AMD64
    return (void *)__readgsqword(0x30);
#endif
}
```

在[程序设计中](../Page/程序设计.md "wikilink")，**缩进风格**（**indent
style**）是管理以表达程序结构的一种约定。本条目主要讨论，例如[C及其后裔](https://zh.wikipedia.org/wiki/C语言 "wikilink")，但这也可以（并经常）适用于大多数其他[编程语言](../Page/编程语言.md "wikilink")（尤其是），其中的则并不重要。缩进风格是[代码风格的一个方面](https://zh.wikipedia.org/wiki/代码风格 "wikilink")。

缩进在大多数编程语言中不是必要条件，而只是作为。不过，缩进有助于更好地向人类阅读者表达程序的结构。尤其是用于澄清[控制流程结构](../Page/控制流程.md "wikilink")（例如条件或循环）与其内部、外部代码之间的关系。不过，部分语言（例如[Python和](../Page/Python.md "wikilink")[occam](https://zh.wikipedia.org/wiki/occam "wikilink")）使用缩进而非大括号或关键词来确定结构，这被称为[越位规则](../Page/越位规则.md "wikilink")。在这种语言中，缩进对编译器或解释器有意义，而不仅仅是清晰度或风格问题。

## 制表符、空格及缩进尺寸

缩进的尺寸通常与风格无关。许多早期程序使用[制表符来缩进](../Page/製表鍵.md "wikilink")，从而简化输入和节约源代码文件的大小。[Unix编辑器通常将制表符视为等同八个字符](https://zh.wikipedia.org/wiki/Unix "wikilink")，而[Macintosh和](https://zh.wikipedia.org/wiki/Mac系列 "wikilink")[Windows环境将它视作四个字符](https://zh.wikipedia.org/wiki/Microsoft_Windows "wikilink")，这使代码在各环境间交换时产生一种混乱。现代的编程编辑器通常可以设置任意的缩进尺寸，并会插入适当的制表符与空格。对[Ruby](../Page/Ruby.md "wikilink")、许多[shell脚本语言和某些形式的](https://zh.wikipedia.org/wiki/shell脚本 "wikilink")[HTML格式](../Page/HTML.md "wikilink")，通常为每个缩进级别使用两个空格。\[1\]

使用制表符还是空格作为缩进字符是编程界的一项持续争论。[傑米·加文斯基等一些程序员认为空格而非制表符有助增加](../Page/傑米·加文斯基.md "wikilink")[跨平台](https://zh.wikipedia.org/wiki/跨平台 "wikilink")[可移植性](https://zh.wikipedia.org/wiki/移植_\(軟體\) "wikilink")。\[2\]而如[WordPress编码规范的作者则认为制表符增加了可移植性](../Page/WordPress.md "wikilink")。\[3\]

## 工具

目前已有许多计算机程序可以自动校正缩进风格（依照程序作者或用户的偏好）以及制表符表示的缩进长度。其中很著名的一个是[`indent`](../Page/Indent_\(UNIX\).md "wikilink")，这个程序包含在许多[类Unix](../Page/类Unix系统.md "wikilink")[操作系统中](../Page/操作系统.md "wikilink")。

在[Emacs中](../Page/Emacs.md "wikilink")，有多种命令可用于自动解决缩进问题。

[Elastic
tabstops是一种需要文本编辑器支持的制表风格](https://zh.wikipedia.org/wiki/Elastic_tabstops "wikilink")，当块中的一行的长度改变时，整个文本块将自动对齐。

### Haskell风格

[Haskell是一种大括号可选的语言](../Page/Haskell.md "wikilink")\[4\]，也就是说，下面的两组代码在语义上是相等的：

``` haskell
braceless = do
  text <- getContents
  let
    firstWord = head $ words text
    bigWord = map toUpper firstWord
  putStrLn bigWord

braceful = do
  { text <- getContents
  ; let
      { firstWord = head $ words text
      ; bigWord = map toUpper firstWord
      }
  ; putStrLn bigWord
  }
```

通常，[procedural](https://zh.wikipedia.org/wiki/过程式编程 "wikilink")
`do`的段落和一般程序文本会省略大括号和分号，但这种风格通常用于由一对括号或大括号组成的列表、记录或其他句法元素，并用逗号或分号分隔。\[5\]

## 其他考虑

### 丢失块踪迹

在某些情况下存在着丢失块边界的轨迹的风险。这通常在包含许多复杂语句的大量代码中看到，这些复合语句嵌套了许多层的缩进。当程序员滚动到一大堆嵌套语句的底部时，他可能已经忘记了哪些控制语句转到哪里。不过，过长的代码也可能有其他原因，
诸如[过于复杂](../Page/循環複雜度.md "wikilink")，面对这个问题的程序员可能会考虑[代码重构以期待它在未来有更好的体验](../Page/代码重构.md "wikilink")。

``` c
for (int i = 0; i < total; i++) {
    foo(bar);
} //for (i)
```

``` c
if (x < 0) {
   bar(foo);
} //if (x < 0)
```

### 声明的插入

在使用标准的[Unix行编辑器](../Page/UNIX.md "wikilink")时，K\&R风格能防止一个常见的错误。在控制语句与循环块的开启大括号之间错误地插入的语句将使循环体变为单次执行。

``` c
for (int i = 0; i < 10; i++)
    whoops(bar);   /* repeated 10 times, with i from 0 to 9 */
{
    only_once();   /* Programmer intended this to be done 10 times */
} //for (i) <-- This comment is no longer valid, and is very misleading!
```

K\&R风格通过将控制语句和开启括号保持在同一行来避免此问题。

**複製建構子**（）是[C++](../Page/C++.md "wikilink")[编程语言中的一種特別的](../Page/编程语言.md "wikilink")[建構子](https://zh.wikipedia.org/wiki/建構子 "wikilink")，習慣上用來建立一個全新的[物件](https://zh.wikipedia.org/wiki/物件_\(電腦科學\) "wikilink")，這個全新的物件相當於已存在物件的[副本](https://zh.wikipedia.org/wiki/物件複製 "wikilink")。這個建構子只有一個參數（引數）：就是用來複製物件的[參照](../Page/參照.md "wikilink")（常用const修饰）。构造函数也可以有更多的参数，但除了最左第一个参数是该类的引用类型外，其它参数必须有默认值。

类的复制构造函数原型通常如下：`Class_name(const Class_name & src);`

一般來說，假如[程式設計師沒有自行編寫複製建構子](../Page/程序员.md "wikilink")，那麼[編譯器會自動地替每一個](../Page/編譯器.md "wikilink")[類別建立一個複製建構子](../Page/类_\(计算机科学\).md "wikilink")；相反地，程式設計師有自行編寫複製建構子，那麼編譯器就不會建立它。

當物件包括[指標或是不可分享的](https://zh.wikipedia.org/wiki/指针_\(信息学\) "wikilink")[參照時](../Page/參照.md "wikilink")，程式設計師編寫顯性的複製建構子是有其必要性的，例如處理[檔案的部分](../Page/電腦檔案.md "wikilink")，除了複製建構子之外，應該還要再編寫[解構子與](../Page/解構子.md "wikilink")[設定運算子的部分](https://zh.wikipedia.org/wiki/C++設定運算子 "wikilink")，也就是[三法則](../Page/三法則.md "wikilink")。

下述代码编译时报错。因为复制构造函数的参数如果是传值，将导致对自身无限递归。

``` cpp
class X；
X(X copy_from_me);
X(const X copy_from_me);
```

调用复制构造函数的情形包括：

1.  用一个对象来初始化正在构造的对象变量；
2.  函数返回一个对象；
3.  函数参数作为对象传值；
4.  抛出一个异常对象；
5.  捕捉一个异常对象；
6.  对象放在大括号中，即{ }。

上述情形未必会调用复制构造函数。因为C++标准允许编译器实现做一些优化。例如：

``` cpp
Class X b=X();
```

Microsoft Visual C++
2010编译器仅执行了一次[缺省构造函数](../Page/缺省构造函数.md "wikilink")，没有执行复制构造函数。

在[计算机科学与](../Page/计算机科学.md "wikilink")[远程通信中](https://zh.wikipedia.org/wiki/远程通信 "wikilink")，**转义字符**放在一个字符序列中时，可对它后续几个字符采取替代解释。转义字符是[元字符的一种特殊情况](https://zh.wikipedia.org/wiki/元字符 "wikilink")。通常，判定某字符是否为转义字符由上下文确定。转义字符即标志着[转义序列开始的那个字符](https://zh.wikipedia.org/wiki/转义序列 "wikilink")。

转义字符的英文为Escape
character。原本是指[ASCII中的十进制](../Page/ASCII.md "wikilink")27，十六进制1B，八进制033所定义的[那个字符](../Page/退出键.md "wikilink")。对应于标准键盘左上角的[ESC键](../Page/退出键.md "wikilink")。老式键盘如果没有ESC键，替代输入是“Ctrl+\[”。在老式的计算机外设控制协议，ASCII码的十进制27这个字符开始的一个字符序列，是外设的控制序列，不能按照这些字符的字面意义解释。后来，就把最初的狭义的Escape
character的含义引申开来，在各种计算机语言与协议中，标志着一个转义序列开始的那个字符，都叫做Escape
character。最常见的一个例子是[C程序设计语言中](https://zh.wikipedia.org/wiki/C程序设计语言 "wikilink")，用反斜线字符“\\”作为转义字符，来表示那些不可打印的ASCII控制符。在[URI协议中](../Page/统一资源标志符.md "wikilink")，[转义字符是百分号](../Page/百分号编码.md "wikilink")“%”。

## 定义

转义字符是很多程序语言、数据格式和通信协议的[形式文法的一部分](../Page/形式文法.md "wikilink")。对于一个给定的[字母表](https://zh.wikipedia.org/wiki/字母表_\(计算机科学\) "wikilink")，一个转义字符的目的是开始一个字符序列，使得转义字符开头的该字符序列具有不同于该字符序列单独出现（没有转义字符开头）时的语义。因此转义字符开头的字符序列被叫做[转义序列](https://zh.wikipedia.org/wiki/转义序列 "wikilink")。一个转义字符可能并没有它自己的意思，因此所有转义序列具有2个或更多字符。

转义序列通常有两种功能。第一个是编码一个句法上的实体，如设备命令或者无法被字母表直接表示的特殊数据。第二种功能，也叫字符引用，用于表示无法在当前上下文中被键盘录入的字符（如字符串中的[回车符](../Page/回车符.md "wikilink")），或者在当前上下文中会有不期望的含义的字符（如[C语言字符串中的双引号字符](https://zh.wikipedia.org/wiki/C语言 "wikilink")"，不能直接出现，必须用转义序列表示）。在后面那种情况，转义序列是一种由转义字符自身和一个被引用的字符组成的一个[二合字母](../Page/二合字母.md "wikilink")（digraph）情形。

### 控制字符

转义字符不属于控制字符；控制字符也不属于转义字符。如果控制字符的定义是非图形的字符，或者对输出设备（打印机、文本终端）有特殊意义的字符，那么针对这些设备的转义字符也是控制字符。但是程序设计用的转义字符是图形字符，因此它们不是控制字符。相反地，大多数ASCII控制字符单独都具有控制功能，因此它们不是转义字符。

## 例子

### ASCII转义字符

ASCII的["escape"字符被用于许多输出设备](../Page/退出键.md "wikilink")，标志着开始一个[转义字符序列](https://zh.wikipedia.org/wiki/转义字符序列 "wikilink")。例如^\[,后跟可打印字符`[2;10H`，将引起[DEC的](https://zh.wikipedia.org/wiki/DEC "wikilink")[VT102终端把光标移动到屏幕的第](https://zh.wikipedia.org/wiki/VT102 "wikilink")2行第10个单元。在1970年代到1980年代的人机界面中，使用Esc键作为转义字符，开始一个转义序列，是非常常见的。但现在一般放弃了这种用法。

### 程序设计与数据格式

许多程序设计语言把双引号字符（`"`）用作字符串的分界符。反斜线（`\`）转义字符提供了两种方式来把双引号字符置入字符串中，或者是使用转义序列`\"`表示单个的`"`字符本身，而不是作为字符串分界符；或者是直接开始字符`"`的16进制编码值的转义序列`\x22`来表示`"`。也可以使用8进制编码值的转义序列，如`\042`。

在[Perl或者](../Page/Perl.md "wikilink")[Python程序设计语言中](../Page/Python.md "wikilink")

:

``` perl
print "Nancy said "Hello World!" to the crowd.";
```

将产生语法错误。而

:

``` perl
print "Nancy said \"Hello World!\" to the crowd.";  ### example of \"
```

将产生期望的输出结果。 另一种可选择的办法是：

:

``` perl
print "Nancy said \x22Hello World!\x22 to them.";  ### example of \x22
```

使用双引号字符编码值的十六进制"x22"转义字符序列。但是，如果其运行在非[ASCII计算机上](../Page/ASCII.md "wikilink")，将不会产生预期的结果。

[C语言](https://zh.wikipedia.org/wiki/C语言 "wikilink"),
[C++](../Page/C++.md "wikilink"),
[Java语言都具有上述两种反斜线转义字符的用法](../Page/Java.md "wikilink")。[PostScript语言与微软的](../Page/PostScript.md "wikilink")[Rich
Text
Format也使用反斜线转义字符](../Page/RTF.md "wikilink")。[quoted-printable编码使用等号](../Page/quoted-printable.md "wikilink")（=）作为转义字符。

[URL与](https://zh.wikipedia.org/wiki/URL "wikilink")[URI使用](https://zh.wikipedia.org/wiki/URI "wikilink")[%](https://zh.wikipedia.org/wiki/% "wikilink")-[转码来表示特殊字符](../Page/百分号编码.md "wikilink")，例如非ASCII字符.
[SGML及其派生的](../Page/SGML.md "wikilink")[HTML与](../Page/HTML.md "wikilink")[XML使用](../Page/XML.md "wikilink")[&字符作为](https://zh.wikipedia.org/wiki/& "wikilink")[字符编码引用的转义字符](https://zh.wikipedia.org/wiki/字符编码引用 "wikilink")。

### Windows命令行

[Windows命令行终端使用](https://zh.wikipedia.org/wiki/cmd.exe "wikilink")[^字符转义那些具有特殊含义的保留字符](https://zh.wikipedia.org/wiki/^ "wikilink")（如：`&
| ( ) < > ^`）。\[1\]
[DOS命令行终端并不支持这一转录](https://zh.wikipedia.org/wiki/COMMAND.COM "wikilink")。

例如，Windows命令行终端中输入这一命令将会产生错误

:

``` dos
echo <wiki>
```

但下述输入将产生正确的期望结果：`<wiki>`

:

``` dos
echo ^<wiki^>
```

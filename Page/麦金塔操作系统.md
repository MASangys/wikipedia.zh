**麦金塔操作系统**（Macintosh operating
systems）是一套运行于苹果[Macintosh系列电脑上的](../Page/Macintosh.md "wikilink")[操作系统](../Page/操作系统.md "wikilink")，包括[macOS](../Page/macOS.md "wikilink")（2012年前称Mac
OS X，2012年-2016年称OS X）、[Mac OS 9](../Page/Mac_OS_9.md "wikilink")、[Mac
OS 8及](../Page/Mac_OS_8.md "wikilink")[System
vX.X這一系列的作業系統](../Page/Mac_OS历史.md "wikilink")。它是一個在商用领域成功的[图形用户界面](../Page/图形用户界面.md "wikilink")，最新版本为[macOS
Mojave](../Page/macOS_Mojave.md "wikilink")。

## 簡介

关于Macintosh是如何被开发和潜在创意的起源，有许多种观点。历史记录里把Macintosh和[施乐公司](../Page/施乐公司.md "wikilink")（Xerox）[帕洛阿尔托研究中心](../Page/帕洛阿尔托研究中心.md "wikilink")（PARC）的Alto计划联系在一起，伊凡·萨瑟兰（Ivan
Sutherland）的画板（Sketchpad）和道格·英格巴特（Doug Engelbart）的在线系统（On-Line
System）对其早期贡献产生了不小的影响。

## 系列操作系统

麦金塔操作系统可被分成两个系列：

  - Classic Mac
    OS，系统搭载在1984年销售的首部Mac与其后代上，采用[Mach作为內核](../Page/Mach.md "wikilink")，在Mac
    OS 7.6.1以前用“System vX.X”来称呼。末代版本是Mac OS 9。
  - 2001年新推出的[macOS](../Page/macOS.md "wikilink")（2012年前称Mac OS
    X，2012年-2016年称OS X），结合[BSD](../Page/BSD.md "wikilink")
    [Unix](../Page/Unix.md "wikilink")、[OpenStep和Mac](../Page/OpenStep.md "wikilink")
    OS
    9的元素。它的最底层建基于[Unix基础](../Page/Unix.md "wikilink")，其核心代码被称为[Darwin](../Page/Darwin_\(操作系统\).md "wikilink")，实行的是部分[开放源代码](../Page/开放源代码.md "wikilink")。

### Classic Mac OS

Classic Mac
OS的特点是完全没有[命令行模式](../Page/命令行.md "wikilink")，它是一个100%的图形操作系统。预示它容易使用，它也被指责为几乎没有[内存管理](../Page/内存管理.md "wikilink")、[协作式多任务和对扩展冲突敏感](../Page/协作式多任务.md "wikilink")。“[功能扩展](../Page/可載入核心模組.md "wikilink")”（Extensions）是扩充操作系统的程序模块，譬如：附加功能性（例如：网络）或为特殊设备提供支持。某些功能擴充套件倾會發生相衝，或-{只}-能按某个特定次序载入。解决Mac
OS的功能擴充套件冲突可能是一个耗时的过程。

Mac OS也引入了一种新形的文件系统，一个文件包括了两个不同的“分支”（forks）。它分别把参数存在“资源分支”（resource
fork），而把原始数据存在“数据分支”（data
fork）裡，这在当时是非常创新的。但是，因为不能识别此系统，这让它与其它操作系统的沟通成为挑战。

最早的Macintosh使用的檔案系統為MFS，由於它屬於平面式（flat）檔案系統，它只提供單一層級的目錄結構。MFS檔案系統很快的在1985年被有B\*樹結構的[HFS取代之](../Page/分層文件系統.md "wikilink")。

### Mac OS X/OS X/macOS

[macOS](../Page/macOS.md "wikilink")（2012年前称Mac OS X，2012年-2016年称OS
X）是Classic Mac OS的继任者。

为了解决以前版本的诸多问题，Mac OS X开始使用基于[BSD
Unix的](../Page/BSD.md "wikilink")[内核](../Page/内核.md "wikilink")，并带来Unix风格的内存管理和[先佔式多工](../Page/先佔式多工.md "wikilink")（pre-emptive
multitasking）。大大改进内存管理，允许同时运行更多软件，而且实质上消除了一个程序崩溃导致其它程序崩溃的可能性。这也是首个包括“命令行”模式的OS，除非执行单独的“终端”（terminal）程序
。但是，这些新特征需要更多的系统资源，按官方的说法早期的Mac OS
X-{只}-能支持[G3以上的新处理器](../Page/G3.md "wikilink")（它在较旧的G3处理器上执行起来比较慢）。

Mac OS X Tiger以前的OS
X有一个[兼容层负责执行老旧的Mac应用程序](../Page/兼容层.md "wikilink")，名为[Classic環境](../Page/Classic環境.md "wikilink")（也就是程序员所熟知的“蓝盒子”\[the
blue box\]）。它把老的Mac OS 9.x系统的完整拷贝作为Mac OS
X里一个程序执行，但执行应用程序的兼容性-{只}-能保证程序在写得很好的情况里在当前的硬件下不会产生意外。2005-2007年苹果开始策划将全线产品更新到intel处理器，抛弃PowerPC处理器，当时的新版Mac
OS X Tiger和Mac OS X Leopard同时支持intel与PowerPC处理器。但其后的Mac OS X Snow
Leopard仅能在intel Core 2或更新的intel处理器上运行，不再支持PowerPC
G3，G4，G5处理器，只可用Rosetta来模拟早期Mac OS X的程序。

2011年以前苹果称自己的OS为“Mac OS X”，发布Mac OS X v10.7后，苹果称其为“OS X
Lion”，其后的系统均改名为“OS X”。OS X Lion及OS X Mountain
Lion只支持2008年后配备intel Core i3以上处理器的Apple计算机，不支持任何PowerPC程序，开放[Mac App
Store](../Page/Mac_App_Store.md "wikilink")。Yosemite以及El
Capitan則提供了平面化的視覺設計。

2016年，[苹果公司将操作系统重新命名为](../Page/苹果公司.md "wikilink")“macOS”。

## 在非苹果电脑上安装Mac OS X

OSx86是把苹果电脑出品的Mac OS X操作系统非官方地移植到普通Intel/AMD架构的计划。

这个计划原来并没有特别的名字，但后来因为OSx86计划网址的出现，使大家都不约而同采用这个名称。

现时，不单有网友成功在[VMware](../Page/VMware.md "wikilink")、[PearPC及](../Page/PearPC.md "wikilink")[VirtualBox模拟器内成功安装](../Page/VirtualBox.md "wikilink")
Mac OS X (Tiger)，在2005年9月，零星把Mac OS安装在一般PC
兼容机的报告也开始出现。不过，要顺利完成安装，使用者还是要把原安装光碟经过第三者的软件修改及配合硬件才能安装。

**點對點隧道協議**（，縮寫為PPTP）是實現[虛擬私人網路](../Page/虛擬私人網路.md "wikilink")（VPN）的方式之一。PPTP使用[传输控制协议](../Page/传输控制协议.md "wikilink")（TCP）建立控制通道來傳送控制命令，以及利用[通用路由封装](../Page/通用路由封装.md "wikilink")（GRE）通道來封裝[点对点协议](../Page/点对点协议.md "wikilink")（PPP）封包以傳送資料。這個協議最早由[微軟等廠商主導開發](https://zh.wikipedia.org/wiki/微軟 "wikilink")，但因為它的加密方式容易被破解，微軟已經不再建議使用這個協議。

PPTP的協定規範本身並未描述[加密或](https://zh.wikipedia.org/wiki/加密 "wikilink")[身份驗證的部份](https://zh.wikipedia.org/wiki/身份驗證 "wikilink")，它依靠[点对点协议](../Page/点对点协议.md "wikilink")（PPP）來實現這些安全性功能。因為PPTP協定內建在[微軟](https://zh.wikipedia.org/wiki/微軟 "wikilink")[視窗系統家族的各個產品中](../Page/視窗系統.md "wikilink")，在微軟[点对点协议](../Page/点对点协议.md "wikilink")（PPP）協議堆棧中，提供了各種標準的身份驗證與加密機制來支援PPTP
\[1\]。
在微軟視窗系統中，它可以搭配[PAP](https://zh.wikipedia.org/wiki/密碼認證協議 "wikilink")、[CHAP](https://zh.wikipedia.org/wiki/CHAP "wikilink")、[MS-CHAP](../Page/MS-CHAP.md "wikilink")
v1/v2或[EAP-TLS來進行身份驗證](../Page/扩展认证协议.md "wikilink")。通常也可以搭配[微軟點對點加密](https://zh.wikipedia.org/wiki/微軟點對點加密 "wikilink")（MPPE）或[IPSec的加密機制來提高安全性](https://zh.wikipedia.org/wiki/IPSec "wikilink")\[2\]。

在[Windows或](https://zh.wikipedia.org/wiki/Windows "wikilink")[Mac
OS平台之外](https://zh.wikipedia.org/wiki/Mac_OS "wikilink")，Linux與FreeBSD等平台也提供開放原始碼的版本。

## 規格

PPTP不是[IETF建議的標準](https://zh.wikipedia.org/wiki/IETF "wikilink")，是由[微軟](https://zh.wikipedia.org/wiki/微軟 "wikilink")、[Ascend
Communications](https://zh.wikipedia.org/wiki/Ascend_Communications "wikilink")（現在属于[Alcatel-Lucent集团](https://zh.wikipedia.org/wiki/Alcatel-Lucent "wikilink")）、[3Com等廠商聯合形成的產業聯盟開發](../Page/3Com.md "wikilink")\[3\]。
1999年7月发布的 RFC 2637是第一個正式的PPTP規格書。

PPTP以[通用路由封装](../Page/通用路由封装.md "wikilink")（GRE）協定向對方作一般的[點對點傳輸](https://zh.wikipedia.org/wiki/點對點 "wikilink")。通過[TCP](../Page/传输控制协议.md "wikilink")1723端口來發起和管理GRE狀態。因為PPTP需要2個網絡狀態，因此會對穿越防火牆造成困難。很多防火牆不能完整地傳遞連線，導致無法連接。
在[Windows或](https://zh.wikipedia.org/wiki/Windows "wikilink")[Mac
OS平台](https://zh.wikipedia.org/wiki/Mac_OS "wikilink")，通常PPTP可搭配[MSCHAP-v2或](https://zh.wikipedia.org/wiki/MSCHAP-v2 "wikilink")[EAP-TLS進行身份驗證](https://zh.wikipedia.org/wiki/EAP-TLS "wikilink")
，也可配合[微軟點對點加密](https://zh.wikipedia.org/wiki/微軟點對點加密 "wikilink")（MPPE）進行連接時的加密。

與PPTP相似的另一種常見的VPN協定是[第二层隧道协议](../Page/第二层隧道协议.md "wikilink")（L2TP）\[4\]。

## 應用

PPTP因為易於設定和是首個支援[微軟](https://zh.wikipedia.org/wiki/微軟 "wikilink")[撥號連線的協議而流行](https://zh.wikipedia.org/wiki/撥號連線 "wikilink")。自[Microsoft
Windows](https://zh.wikipedia.org/wiki/Microsoft_Windows "wikilink") [95
OSR2開始包含PPTP](../Page/Windows_95.md "wikilink")[客戶端](https://zh.wikipedia.org/wiki/客戶端 "wikilink")，但是被限制只可同時存2個向外連線。[Windows
XP的](../Page/Windows_XP.md "wikilink")[遠端桌面連線包含PPTP](https://zh.wikipedia.org/wiki/遠端桌面連線 "wikilink")。

以往，[Linux缺乏完整的PPTP支援](../Page/Linux.md "wikilink")，這是因為MPPE是[软件专利](https://zh.wikipedia.org/wiki/软件专利 "wikilink")。但是，自從在2005年10月28日發佈的Linux
2.6.14起Linux核心提供完整的PPTP支援（包含对自由版本的MPPE支持）。

## 實作

PPTP 是第一個被 Microsoft 撥號網路支援的 VPN 通訊協定。自 Windows 95 OSR2 起，所有的 Microsoft
Windows 版本都內建了 PPTP 客戶端軟體（雖然只能支援最多兩個同時對外的連線）。 在 Mobile 版本上，Microsoft
Windows Mobile 2003 與更新的版本也支援 PPTP。Windows 的路由及遠端存取套件有包含 PPTP 伺服器。
Microsoft 的實作在 MS-CHAP 認證協定內使用了單次 DES 加密，但單次 DES
常被認為無法提供資料足夠等級的安全防護。\[2\]

Windows Vista 與更新版本的 PPTP 開始支援 PEAP。認證機制有 PEAPv0 / EAP-MSCHAPv2 (使用密碼)
還有 PEAP-TLS (智慧卡與憑證)。 Windows Vista 同時停止了 MSCHAP-v1 的認證協定。\[3\]

Linux 的 PPTP 伺服器一般是使用 PoPToP\[4\] 套件加上 PPP 與 MPPE 的 Kernel
Module。PPTP的客戶端程式從 1997\[5\] 就開始提供了，但伺服器部份要等到 Matthew Ramsay 在
1999\[6\]開發的版本從 Moreton Bay 以 GNU GPL
發行後才真正開始有人大量使用。但是，Linux發行版本一開始通常都不包含
MPPE 的部份（因為擔心有專利問題）所以不完整，直到 2005/10/28 發佈的 Linux Kernel 2.6.14 才正式支援
MPPE。 SuSE Linux 10 是第一個有完整可用的 PPTP 客戶端程式的版本。另外還有 ACCEL-PPP 套件: 它提供了
PPTP / L2TP / PPPoE 伺服器\[7\]，可提供 kernel 模式的 PPTP 協定。

OpenBSD 跟 FreeBSD 在 "ports" 套件系統裡提供 PoPToP。 OSX 與 iOS 則提供內建 PPTP 客戶端程式。
OS X Server 內建了 PPTP 服務。 Cisco 與 Efficient Networks 有另外販賣給舊版 Mac OS 使用的
PPTP 客戶端程式。 許多 Android 智慧形手機也支援 PPTP。
